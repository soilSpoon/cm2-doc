Detailed Description

template<class T, size_t M, size_t N>
class cm2::matrix_fixed< T, M, N >

Template matrix with compile-time fixed size.

Template type T can be a POD (unsigned, double, char...) or a user-defined class (with visible default constructor, copy constructor, destructor and assignment operator).

The data are stored contiguously in memory and are column oriented (column-major order). 

   [0] [3] [6] [9]
   [1] [4] [7] [10]
   [2] [5] [8] [11]

This class is simpler and more efficient than dense2D (no smart pointer overhead) but the sizes must be known at compile time and cannot be changed.
Another big difference with dense2D concerns the copy constructors and operators.
The copies here are hard copies, not shallow copies.

Template Parameters
TAny type with default constructor, copy constructor, equal operator and assignment operator.
MThe number of rows of the matrix (fixed unsigned value).
NThe number of columns of the matrix (fixed unsigned value).
See also
dense2D, vector_fixed.

Diagonal iterators

The diagonal_iterator and const_diagonal_iterator can be seen as views to the data of a matrix with a stride equal to M + 1.

This makes them iterate only on the diagonal terms of the matrix.
They are equipped with members such as begin(), end() and size() which make them candidates for most of the cm2::vecscal and cm2::vecvec template functions.
