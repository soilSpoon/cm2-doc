CM2 Math1 SDK Documentation

CM2 Math1 SDK is the basic CM2 mathematical library exporting vector and matrix types.

It contains also numerous template mathematical functions that can be instantiated on the vectors and matrices of Math1 and on others similar math classes (equipped with begin() and end() members).

Just like the STL, CM2 Math1 relies heavily on templates to enable the compiler to generate the most effective code possible.
There is no virtual method no dynamic-binding call and all extensively used functions are inlined.

The math containers can be divided into two main categories:

  • The variable-size containers such as cm2::DoubleVec, cm2::DoubleMat or cm2::DoubleSym.
  • The compile-time fixed-size containers such as cm2::DoubleVec3, cm2::DoubleMat2x2 or cm2::DoubleSym2.

 

Template Mathematical Functions

The template mathematical functions are defined in five namespaces: cm2::vecscal, cm2::vecvec, cm2::matscal, cm2::matvec and cm2::matmat.

Vector-Scalar template functions

Vector-Scalar template functions in the cm2::vecscal namespace.
Example:

cm2::DoubleVec A(100); // Uninitialized vector of size 100.
double s;
cm2::vecscal::copy(0., A); // Set A = 0.
cm2::vecscal::add(1., A); // A += 1.
cm2::vecscal::mult(5., A.sub_vector(0,10)); // Mult by 5. the first ten elements
s = cm2::vecscal::max_value(A); // s = 5.
s = cm2::vecscal::two_norm(A); // s = 18.4390

Note
Variable-size containers use reference-counted smart pointers and heap memory.
Copy and assignment operators share the data (shallow copies).
Fixed-size containers use stack memory and their copy and assignment operators actually copy the data (deep copy).
They are similar to STL array containers (STL C++ 11).
Warning
Variable-size containers are not fully thread-safe because some resources (heap memory) can be shared between several threads.
Copies, assignments and reads are thread-safe, but not writes!
For instance:
cm2::UIntVec V(3, 5); // Vector of three unsigneds initialized to {5, 5, 5}.
#pragma omp parallel for
for (int i = 0; i < 10; ++i)
{
cm2::UIntVec Vi(V); // OK, copy constructors are thread-safe but
// beware that same resource are shared between objects.
unsigned s;
s = vecscal::two_norm(Vi); // OK, thread-safe because only reads in the vector are involved.
Vi[0] = unsigned(i); // A write. Not thread-safe!
// All writes target the same resource: V[0]!
}

Since copies are deep in fixed-size containers, there is no such problem:

cm2::UIntVec3 V(5); // Vector of three unsigneds initialized to {5, 5, 5}.
#pragma omp parallel for
for (int i = 0; i < 10; ++i)
{
cm2::UIntVec3 Vi(V); // Data are copied in each thread.
unsigned s;
s = vecscal::two_norm(Vi); // No problem, thread-safe.
Vi[0] = unsigned(i); // A write but thread-safe because affects only the data in each thread.
}
