CM2 Math1 SDK Release Notes

CM2 Math1 Release 5.6.0

June, 2025.

Caveats

  • Major version. Due to changes in the API, client applications must be recompiled against the new headers.
  • Lib names now end with _56 (Windows) and 5.6.0 with symlinks from 5.6 (Linux, macOS).
  • New builds with clang 20 and gcc 15.1 on Linux (Fedora 42).
  • Builds of Visual Studio 2015 no longer provided.

Improvements

  • The reserve/resize members of the base class cm2::vector_num (on which are based every CM2 Math1 dynamic containers) can now use std::is_trivially_default_constructible (when compiled with C++11 and above) to route more types to fast construction/destruction.

CM2 Math1 Release 5.5.0

June, 2024.

Caveats

  • Major version. Due to changes in the API, client applications must be recompiled against the new headers.
  • Lib names now end with _55 (Windows), 5.5.0 with symlinks from 5.5 (Linux, macOS).
  • Visual Studio 2015 deprecated.

New features

  • New cm2::dense2D::resize_minor (resp. resize_major) to change the number of rows (resp. columns) in a matrix.
  • New cm2::dense1D::resize_major for compatibility with cm2::dense2D.
  • New static functions cm2::matrix_fixed::identity() and cm2::symmetric_fixed::identity() to build identity matrices (ones on the diagonal, zeros elsewhere).
  • New dummy cm2::matrix_fixed::resize (and resize_minor, resize_major) for compatibility with cm2::dense2D.
  • New dummy cm2::vector_fixed::resize (and resize_major) for compatibility with cm2::dense1D.
  • New namespace cm2::matvec_poly for some polymorphic functions on vectors or matrices. To be extended in the future.

Improvements

  • None.

Fixes

  • None.

Breaking changes

  • None.

CM2 Math1 Release 5.3.0

November, 2022.

Caveats

  • Major version. Due to changes in the API, client applications must be recompiled against the new headers.
  • Lib names now end with _53 (Windows).
  • Linux and macOS: dynamic libs now follow the conventional version numbering scheme, have compatibility numbers and come with symlinks. Examples:
    • Linux: libcm2math1.so.5.3.0 (current) with symlinks from libcm2math1.so.5.3 (soname) from libcm2math1.so (base).
    • macOS: libcm2math1.5.3.0.dylib (current) with symlinks from libcm2math1.5.3.dylib (compatibility) from libcm2math1.dylib (base).
  • New macOS arm64 (Apple silicon) and universal (fat libs for both x86-64 and arm64 architectures) builds.
  • Last version with Visual Studio 2013 builds (deprecated and no longer provided in future versions).
  • Last version with Win32 and Linux x86-32 builds (deprecated and no longer provided in future versions).
  • Last version with macOS pure Intel x86-64 builds (deprecated and no longer provided in future versions). Use the universal builds instead.

New features

  • New functions cm2::vecscal::insert and cm2::vecscal::remove to insert and remove a value inside a vector (shifting the subsequent values right or left).
  • New functions cm2::vecvec::insert and cm2::vecvec::remove to insert and remove some values at specific positions inside a vector (shifting the subsequent values right or left).
  • New functions cm2::matvec::insert and cm2::matvec::remove to insert and remove a column inside a matrix (shifting the subsequent columns right or left).
  • New functions cm2::matmat::insert and cm2::matmat::remove to insert and remove some columns at specific positions inside a matrix (shifting the subsequent columns right or left).

Improvements

  • None.

Fixes

  • None.

Breaking changes

  • None.

CM2 Math1 Release 5.1.0

October, 2021.

Caveats

  • Major version. Due to changes in the API, client applications must be recompiled against the new headers.
  • Lib names now end with _51.
  • Win32 and Linux x86 (32-bit) builds are deprecated and will no longer be supported in future versions.
  • Visual Studio 2012 and GCC 6.1 are deprecated and will no longer be supported in future versions (Visual Studio 2010 and GCC 4.8 builds will remain for some times as legacy builds).

New features

  • New cm2::vecvec::diff_max_norm to compute the max norm (i.e. infinite norm) of the difference between two vectors.
  • cm2::vector_fixed, cm2::matrix_fixed, cm2::symmetric_fixed:
    • New swap member.
  • cm2::dense1D, cm2::dense2D, cm2::symmetric_full:
    • New swap member.
  • cm2::matrix_fixed, cm2::symmetric_fixed:
    • New explicit constructors from cm2::vector_fixed (with proper size).
  • New cm2::vecscal::isfinite and cm2::matscal::isfinite to check sanity of vectors and matrices (float and double only).

Improvements

  • cm2::matvec::mult: minors optimisations.

Fixes

  • cm2::symmetric_sparse::JA returned IA instead of JA.

Breaking changes

  • The returned type of many member functions and operators changed from value_type to const_reference.
    This improves noticeably the performance for non-POD types (it saves copies).

CM2 Math1 Release 5.0.0

August, 2020.

Caveats

  • Major version. Due to changes in the API, client applications must be recompiled against the new headers.
  • Lib names now end with _50.
  • Port to GCC 10 and clang 10 with new builds (Linux 32/64 and macOS).
  • Visual Studio 2010 and GCC 4.8 are deprecated and will be no longer supported in future versions.
  • Linux x86 (32-bit) builds shall be soon deprecated.

New features

  • New cm2::symmetric_sparse class with exported pre-instantiations (cm2::UIntSymSparse, cm2::IntSymSparse, cm2::FloatSymSparse, cm2::DoubleSymSparse, cm2::DoubleZSymSparse).
    This class implements a symmetric sparse matrix with compressed row storage (CRS).
    The math libraries cm2::vecscal, cm2::vecvec, cm2::matscal, cm2::matvec and cm2::matmat have been enhanced to support the new class (but some mixed operations between different classes of matrices are still missing).

Improvements

  • None.

Breaking changes

  • cm2::matrix_fixed, cm2::symmetric_fixed:
    • Member data_col (resp. cdata_col) renamed into col_begin (resp. col_cbegin).
  • cm2::dense2D, cm2::symmetric_full:
    • Member data_col (resp. cdata_col) renamed into col_begin (resp. col_cbegin).

CM2 Math1 Release 4.9.0

May, 2019.

Major version. Due to changes in the API, client applications must be recompiled against the new headers.

Caveats

  • Lib names now end with _49.
  • Visual Studio < 2010 and GCC < 4.6 are no longer supported (minimum versions raised to Visual Studio 2010 and GCC 4.8).
  • macOS < 10.8 is no longer supported (minimum deployment target raised to 10.9 Mavericks).

  • New parameter growth_rate (optional with default = 1) in member cm2::dense2d::reserve.
  • New exported classes cm2::UIntVec5, cm2::UIntVec7, cm2::DoubleVec5 and cm2::DoubleVec7.
  • New cm2::vecscal::subtract and cm2::matscal::subtract.

Improvements

  • cm2::matmat::mult and cm2::matmat::transpose_mult: minor OMP improvements.

Fixes

  • cm2::matvec::push_back: several issues when the vector to be appended was empty:
    • If the matrix was not empty upon entry: returned false.
      Now returns true (and still leave the matrix unchanged).
    • If the matrix was empty upon entry: assert violation in DEBUG mode.
      Now the assert violation is silenced.
  • cm2::dense2D::segment_type, cm2::dense2D::const_segment_type, cm2::symmetric_full::segment_type, cm2::symmetric_full::const_segment_type:
    • Issues with member _cols when user side was compiled in DEBUG mode but linked against a CM2 release build.

Breaking changes

  • None.

CM2 Math1 Release 4.8.0

December, 2017.

Major version. Due to changes in the API, client applications must be recompiled against the new headers.

Caveats

  • Lib names now end with _48.
  • New Visual Studio 2017 builds (Win32, Win64, MD, MT).
  • New GCC 5 & 6 builds (Linux86, Linux64).
  • Visual Studio < 2010 and GCC < 4.8 are deprecated and will be no longer supported in future versions.
    In the long term, C++98 will be deprecated. C++11-compliant compilers (for instance Visual Studio >= 2015) will be required to compile the source code of CM2 but also to compile against its API.
  • Windows XP is deprecated and will be no longer supported in future versions.
  • macOS < 10.9 is deprecated and will be no longer supported in future versions (minimum deployment target will raise to 10.9 Mavericks).

Breaking changes

  • cm2::vector_fixed, cm2::matrix_fixed, cm2::symmetric_fixed: constructors with a single initializing value are now qualified explicit.
  • cm2::matvec::Bt_A_B renamed cm2::matvec::Vt_A_V.
  • cm2::matmat::diag_mult: parameter cache_size removed.
  • Default alignment for dynamic arrays is now 8 bytes (instead of 32).

New features

  • segment_iterators and const_segment_iterators for matrices (dynamic and fixed-size): operator*(), operator!= and postfix increment/decrement operators added.
    This makes these column iterators compatible with STL algorithms and enables writing for instance: 
         DoubleMat     M(3, 12, 0.);
     std::for_each(M.seg_begin(), M.seg_end(), [](DoubleMat::segment_type v){ vecscal::copy(v.index(), v); });
     std::for_each(M.seg_begin(), M.seg_end(), [](DoubleMat::segment_type v){ vecio::write(std::cout, v); });

Improvements

  • cm2::matvec::mult and cm2::matvec::transpose_mult now parallelized.
  • cm2::matvec::Vt_A_V (formerly matvec::Bt_A_B) now parallelized and optimized for sparse matrices.

Fixes

  • cm2::matvec::transpose_mult for matrices with separate diagonal storage.

CM2 Math1 Release 4.7.0

June, 2016.

Major version. Due to changes in the API, client applications must be recompiled against the new headers.

Caveats

  • Lib names now end with _47.
  • Visual Studio 2008 is deprecated and will no longer be supported in future versions.

Breaking changes

  • None.

New features

  • New cm2::DoubleZVec, cm2::DoubleZMat, cm2::DoubleZSym for std::complex<double>.
    Fixed-size vectors and matrices for std::complex<double> are not defined because cannot currently be exported by the Windows DLLs.
  • cm2::vecvec::dot, cm2::vecvec::diff_dot, cm2::vecvec::diff_sqr_two_norm, cm2::vecvec::diff_two_norm, cm2::vecvec::cross2, cm2::vecvec::cross_sqr_two_norm, cm2::vecvec::cross_two_norm, cm2::matmat::mult: generalized to work with different base types (such as double and std::complex<double>).
  • New cm2::vecscal::sqr_Herm_norm, cm2::vecvec::Herm_dot and cm2::vecvec::Herm_axpy for complex vectors.
  • New cm2::vecscal::div and cm2::matscal::div (mostly useful for integer types).
  • Access operators (cm2::vector_fixed, cm2::matrix_fixed, cm2::symmetric_fixed, cm2::dense1D, cm2::dense2D, cm2::symmetric_full) and some constructors (cm2::vector_fixed) are now templatized to force their integration into the user's compilands.
    This is to enable assert bound-checkings when the user compile its code in DEBUG mode.
  • New Natvis file for Math1 containers debug visualizers (Visual Studio >= 2012).

Improvements

  • Doc improvements.

Fixes

  • Bug in cm2::matscal::det3 for symmetric matrices.
