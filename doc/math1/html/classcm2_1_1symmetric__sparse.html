Detailed Description

template<class T>
class cm2::symmetric_sparse< T >

Template symmetric sparse matrix with compressed row storage (CRS), resizable, with ref-counting.

Template type T can be a POD (unsigned, double, char...) or a user-defined class (with visible default constructor, copy constructor, destructor and assignment operator).

The data are stored contiguously in memory using a Compressed Row Storage (CRS) scheme.

   [0]     [1] [2]
       [3]     [4]
           [5]    
               [6]

   ia[] = {0 3 5 6 7}
   ja[] = {0 2 3 1 3 2 3}

Just like dense1D from which it inherits many features, symmetric_sparse uses a "smart pointer" scheme to enable different matrices to share the same data and to manage proper memory deallocation.
Copy constructors and operators do not actually copy the data but copy the view to the data.
This is called shallow copy (vs hard copy).
When a symmetric_sparse is copied to another one, the copy is inexpensive and they both share the same data.
A change of an element in one matrix is viewed from the other one.

Warning
symmetric_sparse, as all variable-size containers, are not fully thread-safe because some resources (heap memory) can be shared between several threads.
Copies, assignments and reads are thread-safe, but not writes!
Note
symmetric_sparse uses a smart pointer with reference counting to manage memory deallocation.

TAny type with default constructor, copy constructor and assignment operator.

size_t growth_rate = 1 )
inlineinherited

Manual memory reservation without size change.

The main reason for using reserve() is efficiency.
If you know the capacity to which your vector must eventually grow, then it is usually more efficient to allocate that memory all at once rather than resizing it several times.

The other reason for using reserve() is that you can control the invalidation of iterators.

Returns
True for success.
Note
reserve() has no effect for a value lesser than the current size.

Changes the size of the vector (uninitialized values for the new elements if any).

Elements under min(size(), n) are left unchanged.
If n > size(), new elements between size() and n are uninitialized.
If n > capacity(), a reallocation occurs and iterators are invalid.

Returns
True for success.
Note
When the data are not to be kept, it is a good practice to clear the vector before any resizing.
