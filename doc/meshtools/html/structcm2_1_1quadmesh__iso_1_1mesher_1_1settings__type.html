◆ strict_constraints_flag

bool cm2::quadmesh_iso::mesher::settings_type::strict_constraints_flag

Flag for strict enforcement of hard edges and hard nodes.

When true, an error is raised in the following cases:

  • Two hard nodes are coincident (same coordinates).
  • Two hard edges cut each other.
  • A hard node is inside a hard edge.

When false, the above cases raise only a warning and one of the erroneous item is discarded.

Note
When strict_constraints_flag = false, the lowest edges in data_type::connectB are privileged for enforcement.
When a hard node (resp. edge) is outside the meshed domain only a data_type::CM2_NODE_DISCARDED (resp. data_type::CM2_EDGE_DISCARDED) warning is issued, even in strict-constraints mode.
When strict_constraints_flag = false, the convex-hull is only approximated (the resulting mesh may contain some small non-convexities).
Used in MESH_MODE and CONVEX_HULL_MODE.

Default = true.

◆ subdomains_forcing

int cm2::quadmesh_iso::mesher::settings_type::subdomains_forcing

Forcing mode for intern subdomains.

Possible values are:

  • -1 : All intern subdomains are considered as holes regardless of the orientation of their enclosing boundary.
  • 0 : Intern subdomains are considered as holes when the orientation of their enclosing boundary is opposite to the orientation of the most enclosing domain.
    Otherwise, they are meshed.
  • +1 : All intern subdomains are meshed regardless of the orientation of their enclosing boundary.
Note
Used only in MESH_MODE.

Default = 0.

◆ all_quad_flag

bool cm2::quadmesh_iso::mesher::settings_type::all_quad_flag

Flag to enable/disable the generation of a all-quad mesh (no triangle) as soon as there is parity on the boundaries.

When false, may generate a mixed triangle-quad mesh, even when there is parity on the boundaries.

When true, generate a all-quad mesh as soon as there is parity on the boundaries.

Note
This flag has no effect when there is no parity on the boundaries.
Setting all_quad_flag to true, disables parameter min_Q4_angle_quality (i.e. same as min_Q4_angle_quality = 0).
When this flag is unset or when there is no parity on the boundaries, parameter quadrangle_weight can be used to control the trade-off between meshes with more numerous quads and meshes with more numerous triangles but of better shape quality.
Used in MESH_MODE and in REGULARIZE_MODE.

Default = true.

Examples
tmsh2d01.cpp, tmsh2d03.cpp, tmsh2d34.cpp, and tmsh2di50.cpp.

◆ use_default_background_mesh_flag

bool cm2::quadmesh_iso::mesher::settings_type::use_default_background_mesh_flag

Flag to tell the mesher to use the default background mesh for the interpolation of the metric values.

For expert users only.

The default background mesh is the "front" mesh, i.e. the mesh of the triangulation of the hard nodes without any new node.

When unset, the background mesh used to interpolate the metrics is the mesh currently processed.

Note
Using the current mesh is the fastest mode which indeed gives a similar accuracy.
It is recommended to leave this flag to false.
For expert users only.
Warning
A user-defined background mesh supersedes the default background mesh, even with this flag set.

Default = false.

Examples
tmsh2d03.cpp.

◆ structured_flag

bool cm2::quadmesh_iso::mesher::settings_type::structured_flag

Meshing mode on rectangular domains (unstructured or structured meshes).

Enables structured meshing (grid-like) whenever possible and beneficial.

If true, generates a structured mesh (grid-like) whenever possible, i.e. when:

  • The domain is topologically rectangular (four sharp corners).
  • The discretizations of the four lines match (same number of edges along opposite lines).
  • The domain is roughly convex.
  • There is no isolated hard edge or node (even located outside the domain).
  • There is no repulsive point (even located far outside the domain).
  • The quality of the structured mesh is good enough (in case of a background mesh or target_metric).

If false, fall-back to the unstructured Delaunay-based algorithm (can give also a structured mesh in simple cases).

Note
Used only when refine_flag = true.
Used in MESH_MODE only.

Default = true.

Examples
tmsh2d01.cpp, and tmsh2d34.cpp.

◆ optim_level

unsigned cm2::quadmesh_iso::mesher::settings_type::optim_level

Optimization level between 0 and 10.

Controls the trade-off between speed (low values) and elements quality (high values).
The higher the level, the more aggressive the algorithms.

Note
The algorithms involved here aim at optimizing both the shape quality of the elements and their size quality (length of elements' edges).
Hence, increasing the value of optim_level may lead sometimes to a reduction of shape qualities (min, average or max value) whereas the global quality (which takes into account both types of quality) gets better.
The shape_quality_weight parameter controls the weight of the shape quality in the global quality measure.
Level 0 disables any optimization.

Default = 3

◆ quadrangle_weight

double cm2::quadmesh_iso::mesher::settings_type::quadrangle_weight

Preference for quadrangles versus triangles.

Scalar between 0 and 1 indicating the preference for quadrangles over triangles:

  • For quadrangle_weight = 0, quadrangles are never used.
  • For values between 0 and 1, quadrangles are more and more used, even if this lead to a lesser
    quality of the mesh (a value of 0.5 give no advantage on quads vs triangles).
  • For quadrangle_weight = 1, the number of triangles is minimal (but may be not null).

Default = 0.70 (serious preference for quads). Should not normally be changed.

Note
This parameter is used only when a all-quad mesh cannot be generated.
This parameter is not the ratio between quads and triangles. Furthermore, there is no linearity between this weight and the ratio between quads and triangles.

◆ min_Q4_angle_quality

double cm2::quadmesh_iso::mesher::settings_type::min_Q4_angle_quality

Minimum acceptable angle quality for the quadrangles.

Scalar between 0 and 1 indicating the minimum acceptable angle quality for the quads.

Default = 0 (no minimum angle quality).

Note
This parameter is taken into account in mixed mode only (all_quad_flag = false) and when shell_remeshing_flag = true.
This quality threshold is based on the angle quality of the quads (not the geometric quality which takes the length ratios also into account).
The angle quality is computed as the minimum of the four angles at summits, divided by Pi/2.
The angle quality of a rectangle equals to 1 (perfect) whereas its shape quality is only equal to 1 when the rectangle is a square.
Set min_Q4_angle_quality = 1 to allow rectangles only (quads with right angles only).
Be aware that when boundaries are not straight and convex few perfect rectangles can be generated (mostly triangles).
Used in MESH_MODE only.

◆ length_upper_bound

double cm2::quadmesh_iso::mesher::settings_type::length_upper_bound

Upper bound value for the edge lengths in the final mesh.

The mesher tends to generate meshes with no edge longer than this value (normalized length). But this is not a strict enforcement. Most of the edges will be shorter than this limit, but some may remain somewhat longer.

The default value (1.414) gives the optimal meshes with respect to the size qualities. With this default value (no edge more than 41% longer than the perfect length), the average edge length tends to be 1.

Sometimes it can be useful to limit the length of the edges to a shorter value (usually between 1 and 1.414), and to accept a mean value smaller than 1 (sub-optimal edge qualities).

Note
The value is relative to the perfect edge length.

Default = 1.414

◆ interrupt_hdl

interrupt_handler_type cm2::quadmesh_iso::mesher::settings_type::interrupt_hdl

The user-provided interrupt function.

When this function returns true, the process is stopped.
This can abort the mesh (if the interruption occurred at an early stage), or result in a mesh of poor quality (if the mesher was in the refinement or optimization stage when the interruption occurs).
