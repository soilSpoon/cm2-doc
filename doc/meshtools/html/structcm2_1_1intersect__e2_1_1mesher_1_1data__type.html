enum cm2::intersect_e2::mesher::data_type::warning_type

Warning enums for the mesher.

Non fatal.

If such a warning is raised (returned in the field warning_code), the mesher is not stopped but the user shall check its input and output data for a potential mistake.

High-value warnings have higher precedence than low-value warnings (except CM2_NO_WARNING = 0).
This means than a CM2_REMAINING_INTERSECTIONS won't overwrite a CM2_INTERRUPTION even if some intersections are remaining.

Enumerator
CM2_NO_WARNING 

OK, no warning.

CM2_INTERRUPTION 

The process has been interrupted (by the user through the interrupt handler).

CM2_REMAINING_INTERSECTIONS 

Some intersection(s) remain.

void cm2::intersect_e2::mesher::data_type::set_colors ( const UIntVec & initial_colors)
inline

Re-colorizes the elements.

Simply calls meshtools::change_indices on colors.
Provided for convenience only.

Parameters
[in]initial_colorsThe colors of the initial elements.
Children elements will be colored with same color as their parent.

Example:

intersect_e2::mesher my_fixer;
intersect_e2::mesher::data_type my_data(pos, connectM);
UIntVec initial_colors;
// Do the fixing.
my_fixer.run(my_data);
// Colors of the initial elements (user-defined).
// Three colors only: 0 to elements 0..99, 2 to elements 100..199 and 1 for the others.
initial_colors.resize(connectM.cols(), CM2_NONE);
for (size_t i = 0; i < 100; ++i)
initial_colors[i] = 0;
for (size_t i = 100; i < 200; ++i)
initial_colors[i] = 2;
for (size_t i = 200; i < connectM.cols(); ++i)
initial_colors[i] = 1;
// Apply these three colors to the new mesh.
// Children of elements 0..99 (resp. 100..199 or 200+) will be assigned color 0 (resp. 2 or 1).
mydata.set_colors(initial_colors);

◆ set_colors_by_parts()

int cm2::intersect_e2::mesher::data_type::set_colors_by_parts ( bool reordering_flag)

Re-colorizes the elements with their part id.

All elements of first part will be colored with 0, elements of second part will be colored with 1 and so on.

A reordering of elements by increasing part ids is possible (through reordering_flag).

Parameters
reordering_flagFlag to reorder the elements by increasing part id (true) or leave then in natural order (unchanged elements of part #0, unchanged elements of part #1..., new elements of part #0, new elements of part #1...)
xparts is updated accordingly.
Returns
Error code. Negative value in case of error (xparts not sorted or memory error).
Note
Requires xparts to be defined otherwise all elements will be recolored with 0.
With a reordering (reordering_flag = true), unchanged edges and new edges are interleaved in data_type::connectM (the unchanged edges are no longer the data_type::nefs_in_good first ones).

◆ pos

DoubleMat cm2::intersect_e2::mesher::data_type::pos

The coordinate matrix (column oriented).

Node i has coordinates in column pos.col(i).

Upon entry, this matrix must contains the coordinates of all the points (i.e. nodes referenced in connectM).

It may also contain other unreferenced points.

Upon exit, new nodes (if needed) are appended at the back of the matrix and the indices in the final mesh (connectM) refer to column numbers in this matrix.

Mode = IN-OUT.

Note
The matrix must have 2 rows (2-D) or 3 rows (3-D).
Examples
tmsh2di50.cpp.

◆ connectM

UIntMat cm2::intersect_e2::mesher::data_type::connectM

The connectivity of the edge mesh.

Intersecting edges (i.e. within the settings_type::tolerance distance of another edge or a hard node) are removed from this array and the intersected edges are split.

Upon exit, only non-intersecting edges are left in this array.

Note
The mesh needs not to be continuous or in a single piece. It can contains several subdomains and can be non manifold.
The values are node indices and refer to columns in the coordinate matrix pos.
Upon exit, the good edges (i.e. non-intersecting) are relocated to the heading columns, from column 0 to column nefs_in_good (past-the-end).
The new split edges are appended to the back, from column nefs_in_good to column connectM.cols() (past-the-end).
The new split edges may need a remeshing (see cm2::meshtools1d::remesh_lines) to improve elements length.

Mode = IN-OUT.

Examples
tmsh2di50.cpp.

◆ xparts

UIntVec cm2::intersect_e2::mesher::data_type::xparts

The past-the-end column indices delimiting parts (subdomains) in connectM.

When xparts.size() > 1 (i.e. defining at least two parts) a cross analysis can be performed (assuming settings_type::analysis_code = 0).
This saves CPU time when we know that each part individually is fine but we want to fix only the intersections between the parts.
In addition, this prevents close nodes or small elements in each part to be considered as pathology and modified.

Note
Upon exit, as pathological elements are removed from connectM and new elements are appended, these indices do not refer any more to ends of parts.
To reorder the elements by increasing part ids and retrieve the new past-the-end column indices use set_colors_by_parts.
Precondition
Must be sorted ascendingly (if not empty).
The last index (if not empty) must be lesser or equal to connectM.cols().
Elements from xparts.back() to connectM.cols() are excluded from the analysis when settings_type::anaysis_code < 2.
Note
Discarded when settings_type::anaysis_code >= 2 (full analysis).

Mode = IN.

◆ colors

UIntVec cm2::intersect_e2::mesher::data_type::colors

The IDs of the parent edge in the initial mesh.

This vector gives the column IDs in the initial connectM matrix of the edge
from which the new edge stems from.

Note
Non-intersecting edges are not modified but can be relocated to a different column of the new connectM matrix.
The "color" of a non-intersecting edge is the column ID of this edge in the initial mesh.
Intersecting edges are split into several new sub-edges (children).
Each child is affected with the same "color" which is the column ID of the parent edge in the initial connectM matrix.
This vector is useful to transfer properties from the initial mesh to the new mesh.
Postcondition
colors.size() == connectM.cols()

Mode = OUT.
