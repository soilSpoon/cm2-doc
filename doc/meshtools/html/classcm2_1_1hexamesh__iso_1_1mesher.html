Detailed Description

Unstructured hex-dominant isotropic mesher.

Instances of this class mesh a 3-D domain with hexahedrons, wedges, pyramids and tetrahedrons.

Domains are defined by boundary meshes (all-quadrangle, all-triangle or mixed) and possibly some embedded nodes, edges and faces.

It can also be used as an optimizer of an already existing mesh.

All the data to be passed to the mesher are gathered in a single structure of type hexamesh_iso::mesher::data_type.

Note
Upon entry, the boundary faces in data_type::connectB need to be oriented the same way in the case of several subdomains or when internal closed surfaces (such as cavities) are present.
Boundary quadrangles are guaranteed to be all recovered and present in the final mesh only if they are flat (even when settings_type::hard_quads_enforcement_flag = true).
In difficult cases, warped quads may be split into two triangles (even when settings_type::hard_quads_enforcement_flag = true).
Boundary triangles are always recovered and present in the final mesh (assuming they do not intersect with any other hard entity).
Thread safe.
