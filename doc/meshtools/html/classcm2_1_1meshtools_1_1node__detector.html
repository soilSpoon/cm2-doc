Detailed Description

Node detector.

A node detector can be used to find the closest node to a point or the nodes inside a box. Internally, it uses a balanced 2-D or 3-D bisection tree to make the searches efficient.

Warning
After construction or reinitialization, the number of rows in the coordinate matrix pos must not be changed.
Once nodes have been inserted in the detector (through construction, reinitialization or simple insertion), their coordinates in matrix pos should not changed (but new coordinates may be back-pushed).
Examples
tmsh2di50.cpp.

unsigned cm2::meshtools::node_detector::find_close ( const DoubleVec2 & P) const

A node close to a given point (not always the closest).

Parameters
[in]PThe point for which a close node is to be searched.
Returns
The index of a close node (CM2_NONE if the detector is empty). Returned value = -k => the k-th argument had an illegal value.
Note
If the detector has been initialized with a 3-D coordinate matrix, find a close point to {P[0], P[1], 0}.
Faster than find_closest but doesn't return always the closest node.

UIntVec & nodes ) const

Gets the nodes located in a given 2-D box.

Push back into vector nodes the IDs of the recorded nodes found strictly inside a specific box. The box is oriented as XY and is defined by the MinBox and MaxBox coordinates.

Parameters
[in]MinBoxThe min summit of the box.
[in]MaxBoxThe max summit of the box.
[in,out]nodesThe vector in which the found nodes are appended.
Note
If the detector has been initialized with a 3d-coordinate matrix, zero is used for the third coordinate of MinBox and MaxBox.
