Detailed Description

Unstructured anisotropic quadrangle mesher (all-quad and mixed triangle/quad mesh generator).

2-D mesh with anisotropic metrics on boundary nodes.

Instances of this class generate all-quad or mixed quad-dominant meshes on 2-D domains defined by boundary edges and possibly some embedded nodes and edges.

They can also be used as shape optimizers for already existing meshes.

All the data to be passed to the mesher are gathered in a single structure of type quadmesh_aniso::mesher::data_type.

In order to generate a all-quad mesh, this mesher needs all boundary lines to be meshed with an even number of edges, otherwise only a mixed triangle-quad mesh is generated (quad dominant mesh).

Parity along the boundaries can be achieved for instance by using the force_even flag in the meshtools1d mesh generation functions (such as meshtools1d::mesh_straight).

Example: Mesh of a square

INPUT:
                                            3       6       2
   data.pos = [0 2 2 0 1 2 1 0               *------*------*
               0 0 2 2 0 1 2 1]              |             |
                                             |             |
   data.connectB = [0 4 1 5 2 6 3 7          |             |
                    4 1 5 2 6 3 7 0]       7 *             * 5
                                             |             |
                                             |             |
                                             |             |
                                             *------*------*
                                            0       4       1


quadmesh_aniso::mesher   my_mesher;
my_mesher.run(data);


OUTPUT:
                                            3       6       2
   data.pos = [0 2 2 0 1 2 1 0 1             *------*------*
               0 0 2 2 0 1 2 1 1]            |      |      |
                                             |      |      |
   data.connectM = [0 7 4 8                  |      |8     |
                    4 8 1 5                7 *------*------* 5
                    8 6 5 2                  |      |      |
                    7 3 8 6]                 |      |      |
                                             |      |      |
                                             *------*------*
                                            0       4       1
Note
Upon entry, the boundary edges in data.connectB need to be oriented the same way in the case of several subdomains or when internal closed lines (such as holes) are prescribed.
However, edges don't need to be consecutive in the matrix.
Upon exit, the elements are always oriented counter clock-wise, i.e. with the normal = Oz.
Thread safe.
Warning
The third coordinates in matrix data.pos is discarded for all points.
That means that all points are projected onto the Z=0 plane (even for points not referenced in data.connectB or data.isolated_nodes).
