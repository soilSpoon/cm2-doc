The data type for surfmesh_q4::mesher.

◆ warning_type

enum cm2::surfmesh_q4::mesher::data_type::warning_type

Warning enums for the mesher.

Non fatal.

If such a warning is raised (returned in the field warning_code), the mesher is not stopped but the user shall check its input and output data for a potential mistake.

High-value warnings have higher precedence than low-value warnings (except CM2_NO_WARNING = 0).
This means than a CM2_SHAPE_QUALITY_WARNING won't overwrite a CM2_INTERRUPTION even if some elements have a bad quality (Qs < 0.01).

Enumerator
CM2_NO_WARNING 

OK, no warning.

CM2_INTERRUPTION 

The process has been interrupted (by the user through the interrupt handler).

CM2_FAILED_SURFACES_WARNING 

One or several surfaces could not be meshed.

CM2_SHAPE_QUALITY_WARNING 

One or several elements have a bad quality (Qs < 0.01).

◆ vertex_H

DoubleVec cm2::surfmesh_q4::mesher::data_type::vertex_H

The mesh sizes (isotropic) on vertices.

vertex_H[i] = the mesh size on vertex with CAD index i (following the CAD indexing scheme).

Note
With OCCT, we use the CAD indexing scheme given by TopTools_IndexedMapOfShape.
Example: 
            TopTools_IndexedMapOfShape       vertices;
            TopExp::MapShapes(root, TopAbs_VERTEX, vertices);     // Index all vertices of root consecutively starting from 1.
            unsigned i = vertices.FindIndex(some_vertex);         // `i` is the index we use to refer to `some_vertex`.
As OCCT follows a 1-base indexing scheme, vertex_H[0] is discarded.
Only strictly positive values are considered with a minimum value of settings_type::min_h.
Vertices with CAD index greater than or equal to vertex_H.size() are assigned a default mesh size derived from the lower precedence mesh sizes (curve_H, surface_H or settings_type::target_h).
vertex_H has the highest precedence over all user-defined mesh sizes. The precedence order is:
  1. vertex_H (highest precedence)
  2. curve_H
  3. surface_H
  4. settings_type::target_h (lowest precedence)
The mesher may use reduced mesh sizes in all directions (isotropic) or in selected directions (anisotropic) to honor the settings_type::max_chordal_error criterion.

Mode = IN.

◆ curve_H

DoubleVec cm2::surfmesh_q4::mesher::data_type::curve_H

The mesh sizes (isotropic) on curves.

curve_H[i] = the mesh size on curve with CAD index i (following the CAD indexing scheme).

Note
With OCCT, we use the CAD indexing scheme given by TopTools_IndexedMapOfShape.
Example: 
            TopTools_IndexedMapOfShape       curves;
            TopExp::MapShapes(root, TopAbs_EDGE, curves);      // Index all curves of root consecutively starting from 1.
            unsigned i = curves.FindIndex(some_curve);         // `i` is the index we use to refer to `some_curve`.
The CAD index is also the color of the curve.
As OCCT follows a 1-base indexing scheme, curve_H[0] is discarded.
Only strictly positive values are considered with a minimum value of settings_type::min_h.
Curves with CAD index greater than or equal to curve_H.size() are assigned a default mesh size derived from the lower precedence mesh sizes (surface_H or settings_type::target_h).
The mesh size on a curve is also transferred to its (two) extremity vertices, unless valid mesh sizes are provided for them in vertex_H.
The mesher may use reduced mesh sizes in all directions (isotropic) or in selected directions (anisotropic) to honor the settings_type::max_chordal_error criterion.

Mode = IN.

◆ surface_H

DoubleVec cm2::surfmesh_q4::mesher::data_type::surface_H

The mesh sizes (isotropic) on surfaces.

surface_H[i] = the mesh size on surface with CAD index i (following the CAD indexing scheme).

Note
With OCCT, we use the CAD indexing scheme given by TopTools_IndexedMapOfShape.
Example: 
            TopTools_IndexedMapOfShape       surfaces;
            TopExp::MapShapes(root, TopAbs_FACE, surfaces);    // Index all surfaces of root consecutively starting from 1.
            unsigned i = surfaces.FindIndex(some_surface);     // `i` is the index we use to refer to `some_surface`.
The CAD index is also the color of the surface.
As OCCT follows a 1-base indexing scheme, surface_H[0] is discarded.
Only strictly positive values are considered with a minimum value of settings_type::min_h.
Surfaces with CAD index greater than or equal to surface_H.size() are assigned the ultimate default mesh settings_type::target_h.
The mesh size on a surface is also transferred to all its boundary curves and vertices, unless valid mesh sizes are provided for them in curve_H or vertex_H.
The mesher may use reduced mesh sizes in all directions (isotropic) or in selected directions (anisotropic) to honor the settings_type::max_chordal_error criterion.

DoubleMat cm2::surfmesh_q4::mesher::data_type::pos

The coordinate matrix (column oriented).

Node i has coordinates in column pos.col(i).

The dimensions of this matrix are 3 x nods.

Note
When the CAD indexing scheme is 1-base (as with OCCT), there is a left one-shift in the indices.
The coordinates of CAD vertex with id = i is at column i-1 in this pos matrix.

◆ connectM

UIntMat cm2::surfmesh_q4::mesher::data_type::connectM

The connectivity of the elements.

Dimensions are 4 x nefs when settings_type::high_order_type = 0, 8 x nefs when settings_type::high_order_type = 1 and 9 x nefs when settings_type::high_order_type = 2.

The leading part of this matrix (from col 0 to col nefs_Q - 1) is the connectivity of the quadrangle elements.

The tailing part of this matrix (from col nefs_Q to col nefs - 1) is the connectivity of the triangles elements, completed with CM2_NONE values (i.e. unsigned(-1)).

Note
The values are node indices and refer to columns in the coordinate matrix pos.

Mode = OUT.

◆ colorsE

UIntVec cm2::surfmesh_q4::mesher::data_type::colorsE

The coloring of the edges (following the CAD indexing scheme).

Each edge of connectE is assigned a color (from 1 to nbr_curves) corresponding to the CAD id of its curves.

Postcondition
colorsE.size() = connectE.cols().
Note
With OCCT, we use the CAD indexing scheme given by TopTools_IndexedMapOfShape.
Example: 
            TopTools_IndexedMapOfShape       curves;
            TopExp::MapShapes(root, TopAbs_EDGE, curves);      // Index all curves of root consecutively starting from 1.
            unsigned i = curves.FindIndex(some_curve);         // `i` is the index we use to refer to `some_curve`.
The CAD index is also the color of the curve.
