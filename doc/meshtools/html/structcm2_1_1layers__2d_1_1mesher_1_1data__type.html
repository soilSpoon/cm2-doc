Detailed Description

The data type for layers_2d::mesher.

This structure is used to gather all the input and output data of the member function layers_2d::mesher::run.

Examples
tmsh2db02.cpp, and tmsh2db06.cpp.

Constructor.

Constructs the data with a coordinate matrix, a boundary edges matrix (shallow copy) and a depth defined on each hard edge.

This is the simplest way to create the data to exchange with the mesher.

Parameters
[in]PThe coordinate matrix. New nodes are appended by the mesher.
[in]connectE2The connectivity matrix for the boundary mesh.
[in]depthThe user defined depth defined on each hard edge.
Note
After the meshing process, you must get the new coordinate matrix from the field pos, and the connectivity matrix connectM as well or use the extract function.
Warning
Upon exit, the matrix data.pos is resized to two rows. The third coordinate (i.e. Z) is lost for all points.

◆ pos

DoubleMat cm2::layers_2d::mesher::data_type::pos

The coordinate matrix (column oriented).

Node i has coordinates in column pos.col(i).

Upon entry, this matrix must contains the coordinates of all the points (i.e. nodes referenced in connectB0).

It may also contain other unreferenced points.

Upon exit, the new nodes are appended at the back of the matrix and the indices in the final mesh (connectM) refer to column numbers in this matrix.

Mode = IN-OUT.

Warning
On exit, the matrix is resized to two rows. The third coordinate (Z) is lost for all points.
Examples
tmsh2db02.cpp, and tmsh2db06.cpp.

◆ connectB0

UIntMat cm2::layers_2d::mesher::data_type::connectB0

The hard edges (boundary edges + internal hard edges).

This connectivity matrix must define a valid and closed CM2_EDGE2 mesh for the external boundary of the domain to be meshed.

The orientation of these boundary edges usually needs to be similar (either all CW or all CCW), except when there is only a single external contour.

Internal hard edges can be added to the boundary mesh.

In order to define holes inside the domain, the boundary mesh of the holes must be closed and oriented oppositely to the external boundary mesh.

Note
The values are node indices and refer to columns in the coordinate matrix pos.

Mode = IN.

Examples
tmsh2db02.cpp, and tmsh2db06.cpp.

◆ depth

DoubleVec cm2::layers_2d::mesher::data_type::depth

User-defined edges depth.

depth[k] is the depth for the k-th edge (k-th column in connectB0).

Set depth[k] = 0 to prevent the generation of any layer above boundary element k.

Note
If depth.size() > connectB0.cols(), extra depths are ignored.
If depth.size() < connectB0.cols(), missing depth are set to 0 (i.e. no layer will be generated above these edges).

Mode = IN.

Examples
tmsh2db06.cpp.

◆ connectM

UIntMat cm2::layers_2d::mesher::data_type::connectM

The connectivity of the generated elements (column oriented).

The dimension of this matrix is 4 X nefs.

The leading part of this matrix (from col 0 to col nefs_Q4 - 1) is the connectivity of the quadrangle elements. The tailing part of this matrix (from col nefs_Q4 to col nefs - 1) is the connectivity of the triangles elements. The fourth node ID in this part of the matrix is always CM2_NONE (i.e. unsigned(-1)).

Note
The values are node indices and refer to columns in the coordinate matrix pos.
The number of rows is always equal to 4, even if the layer contains only triangles. In this case, the fourth row contains only CM2_NONE values.

Mode = OUT.

Examples
tmsh2db02.cpp, and tmsh2db06.cpp.

The connectivity of the final layers contour, i.e.

intern contour (column oriented).

The dimension of this matrix is 2xNEFS.

Note
The values are node indices and refer to columns in the coordinate matrix pos.
Use connectB1 as input of cm2::triamesh_iso::mesher or cm2::quadmesh_iso::mesher to generate the rest of the mesh in the domain.

Mode = OUT.
