Transforms a series of node indices into a mesh of CM2_EDGE2 elements.

Parameters
[in]indicesThe indices of the nodes (geometrically consecutive).
[in,out]connectEThe connectivity matrix (new columns are appended).
The matrix is resized to 2x(N0+N-1), with N0 = number of columns on input and N = indices.size()
connectE(0, N0+j) = indices[j];
connectE(1, N0+j) = indices[j+1];
Returns
Error code. Negative value in case of error.
Returned value = -k => the k-th argument had an illegal value.
Note
Does not create any new node.
Precondition
connectE.rows() == 2 or connectE is empty.
indices.size() >= 2
Examples
tmsh2d01.cpp, tmsh2d03.cpp, tmsh2d34.cpp, tmsh2da51.cpp, tmsh2da56.cpp, tmsh2di07.cpp, and tmsh3di01.cpp.

◆ convert_into_high_order() [1/2]

int cm2::meshtools1d::convert_into_high_order ( DoubleMat & pos,
const UIntMat & connectL,
unsigned Ne,
UIntMat & connectQ )

Converts E2 elements into higher degree elements (E3, E4 and over), creating new high-order nodes.

Nodes are locally numbered as follow:

  • 0 and 1 : initial summits
  • 2 to 2 + Ne - 1 : new nodes inside edge

Example: Conversion of a E2 into a E4. 

INPUT:

      *--------------|--------------*
      0              1              2

   pos = [0  3  6  
          0  0  0                         (coordinates stored column-wise)
          0  0  0]

   connectivity = [0  1                   (connectivity stored column-wise)
                   1  2]                    

   Ne  = 2                                (2 more nodes on each edge)


OUTPUT:                          

   pos = [0  3  6  1  2  4  5  
          0  0  0  0  0  0  0             (coordinates stored column-wise)
          0  0  0  0  0  0  0]

   connectivity = [0  1                   (connectivity stored column-wise)
                   1  2
                   3  5             
                   4  6]                    

      *----|----|----*----|----|----*
      0    3    4    1    5    6    2
Parameters
[in,out]posThe coordinate matrix. New high-order nodes are appended.
[in]connectLThe connectivity of the linear mesh (dimensions 2 x NE).
[in]NeThe number of nodes to add on each edge (any value >= 0)
[out]connectQThe connectivity of the high-order mesh. Any input data is lost.
Returns
Error code. Negative value in case of error.
Returned value = -k => the k-th argument had an illegal value.
Note
New nodes are created following the order of the edges in connectL. First the new node(s) for edge #0 (column 0 in connectL), then for edge #1 (column 1 in connectL) and so on.
The internal nodes are created on the straight line between the initial summits of each edge.
To get curved elements, you need to map these nodes to the "true" parametric line.
New nodes are equally spaced between the initial summits of each edge.
Precondition
All the values in the connectivity matrix connectE are smaller than pos.cols().
connectL.rows() == 2 or empty.
Postcondition
connectQ.rows() == 2 + Ne.
connectQ.cols() == connectL.cols().
See also
meshtools1d::convert_into_linear.
meshtools2d::convert_into_high_order, meshtools3d::convert_into_high_order.

Converts in-place high-order elements (CM2_EDGE3 or higher) into linear elements (CM2_EDGE2).

Very simple function (just resizing the matrix, time complexity = O(1)) provided for similarity with meshtools2d::convert_into_linear and meshtools3d::convert_into_linear.

Parameters

Postcondition

Creates a straight line mesh from a node and a translation (2-D version).

Example: 

INPUT :

   pos   = [0
            0
            0]
   N0    = 0
   T     = [1.5 3 0]
   N     = 3


OUTPUT :
                                  
   pos = [0  0.5  1  1.5
          0   1   2   3          (stored column-wise)
          0   0   0   0 ]

   indices = [0 1 2 3]       

   3 elements :


                   3        
                  *            _
                 /             /|
                /             /
              2/             /
              *             /
             /             /  
            /             /   T
           /             / 
         1*             /
         /             /   
        /             /
       /             /
   P0 *             /
      0

Can be used in conjunction with meshtools1d::indices_to_connectE2 to convert indices into a connectivity matrix.

Parameters
[in,out]posThe coordinate matrix. New points are appended.
[in]N0The index of the starting node.
[in]TThe translation vector.
[in]NEThe number of elements to divide the translation vector.
[in,out]indicesThe indices of the nodes (NE + 1 indices are appended).
The index N0 is first appended, then the N new nodes index.
Returns
Error code. Negative value in case of error.
Returned value = -k => the k-th argument had an illegal value.
Note
A null translation vector causes the elements to be degenerated.
Precondition
N0 < pos.cols()
NE > 0
pos.rows() == 2 or pos is empty.
See also
indices_to_connectE2, mesh_straight, extrude_rotate.

DoubleVec & H,
UIntMat & connectE,
double target_h,

Meshes a parametric 3-D curve (isotropic version).

This function is a template function to allow the user to mesh her/his own parametric curves.

Another advantage of the template function is that an experienced user can freely adapt it to his/her specific needs.

Template Parameters
CurveParametric curve.

The Curve template class should be equipped with the following members:

int get_3D_coordinates (const DoubleVec& pos1d, DoubleMat& pos3d) const;
int get_1D_coordinates (const DoubleMat& pos3d, DoubleVec& pos1d) const;
int get_tangents (const DoubleVec& pos1d, DoubleMat& T) const;
int get_curvatures (const DoubleVec& pos1d, DoubleVec& C) const;
cm2::DoubleMat
dense2D< double > DoubleMat
cm2::DoubleVec
dense1D< double > DoubleVec

The Curve::get_3D_coordinates function computes the 3-D coordinates of a set of 1-D points located along the reference curve (U).
The 3-D coordinates associated to the point at index j of pos1d must be returned in column j of pos3d.
This function should return zero when successful and a negative value (-1 for instance) when failed.

The Curve::get_1D_coordinates function is the reciprocal function of the previous one.
It gives the 1-D reference coordinate (U) of a set of 3-D points.
The reference coordinate of the point at index j of pos3d must be returned at index j of pos1d.
Note that this function is called only for the initial nodes if pos1d is empty upon entry, or for the high-order nodes (when high_order_type > 0).
This function should return zero when successful and a negative value (-1 for instance) when failed.

The Curve::get_tangents function computes the tangent on the curve at a set of points given by their reference coordinates (U). These tangents must not be normalized.
They are the mere derivatives of the curve with respect to the curvilinear parameter U.
The tangent at the point at index j of pos1d must be returned in column j of T.
This function should return zero when successful.

The Curve::get_curvatures function computes the curvatures on the curve at a set of points given by their reference coordinates U.
The curvature C is defined as: 

       ||D2P/Du2 x DP/Du||     ||DT/Du x T||
   C = -------------------  =  -------------
            ||DP/Du||3             ||T||3

Norm of cross-product between the derivative of T (local tangent) with respect to U (i.e. second derivative of P(u) with respect to u2) and the normalized tangent T to the curve.
This function should return zero when successful and a negative value (-1 for instance) when failed.
Should you leave the implementation of this member empty (returning -1 for instance), approximative curvatures computed from variations of the tangents will be used instead.

It is the responsibility of the user to provide this template function with a coherent Curve class.
That is, a class for which get_1D_coordinates and get_3D_coordinates do reciprocal computations, and get_tangents computes the derivative of the curve with respect to the reference parameter, and get_curvatures computes the C data as defined above.
This template function does not control this coherence.

Should the parametric curve be not regular on the domain to be meshed (i.e. with some singularity or degeneracies leading to null tangent vectors) this template functions will use default metrics instead.
This may lead however to locally poor mesh quality.

[in,out]posThe 3-D coordinates of the nodes.
New nodes created along the curve are appended (linear nodes first, then high-order nodes if any).
[in]nodesThe initial node IDs along the curve, sequentially from lowest curvilinear coordinates to highest curvilinear coordinates.
Usually the two extremities of the curve.

[in,out]HUpon entry: the requested mesh sizes at nodes (in same order). Upon exit: the sizes for the new linear nodes are appended (the sizes for the high-order nodes are set to 0).

[out]connectEThe edges as a connectivity matrix (with 2 + high_order_type rows), consecutively from the first node to the last node (in nodes).
[in]target_hThe requested mesh size when H is empty upon entry or when there is only two nodes.
[in]force_evenFlag to force the number of created edges to be even (useful for subsequent all-quad meshing).
[in]min_nThe minimum number of edges that should be created along the curve.
[in]max_nThe maximum number of edges that should be created along the curve.
[in]max_chordal_errorMaximum chordal error allowed:
  • If max_chordal_error > 0, the chordal error is considered as absolute.
    The distance between the elements and the curve will be bounded by max_chordal_error.
  • If max_chordal_error < 0, the chordal error is considered as relative with respect to the curvature radius.
    The distance between the elements and the curve will be bounded by - max_chordal_error * curvature radius. The chordal error control is disabled when max_chordal_error = 0 or +-DBL_MAX or when chordal_control_type = 0.

  • 2: Convert to CM2_EDGE4...

[in]high_order_mode[OPT] Method of generation for the high-order nodes (HON):
  • 0: The HON are interpolated between the linear 3-D nodes.
    These nodes are not on the curve (straight elements).
    The reference coordinates are approximative (middles between those of the linear nodes).
    This method is the fastest method.
  • 1: The HON are on the curve and their reference coordinates are the middles between those of the linear nodes but may not be equidistant between the linear 3-D nodes.
    The speed of this method is between methods 0 and 2.
  • 2: The HON are on the curve and equidistant between the linear 3-D nodes but may not have reference values in the middle between the linear nodes.
    This method is the slowest method. Relevant only when high_order_type > 0.

[in]max_bgm_remeshings[OPT] Max number of remeshing iterations to generate a background mesh.

The spline is a G1 cubic curve defined by two summits (ib and ic) and two tangents.
The first and the last points (ia and id) are used to define the tangents at the beginning and at the end of the spline.

These tangents are computed as follow:

Changes the orientation of the edges so that all elements have similar orientation as a reference element.

The algorithm involved is a neighboring search starting from element ref_element.

If code == 0, only the elements in the same subdomain(1) as element ref_element will be visited. Other elements will remain unchanged.
If code != 0, all elements will be visited. Elements in other subdomains than that of ref_element, if any, will be reoriented as the first (lowest ID) element in each subdomain.

Note
A subdomain is a domain's subset of connected elements.

The neighbors matrix is defined by:

neighbors(i,j) = neighbor of element j by the boundary i with the local node and boundary IDs defined as follow:

         N0             N1

[in]ref_elementThe index of the reference element in connect.
[in]codeThe code to tell whether all elements must be processed (code != 0), or only the elements among the subdomain of ref_element (code == 0).

Remeshes lines, connected or not, with possibly user-defined hard nodes and colors.

Similar to CM2 SurfRemesh T3/Q4 but for edge meshes.

Parameters
[in,out]posThe coordinate matrix (2D or 3D). Coordinates for new nodes are appended upon exit.
[in,out]connectEThe connectivity of the mesh (CM2_EDGE2 only).
Initial mesh upon entry, new mesh upon exit.
[in]target_hThe default size for the new edges.
This value is superseded locally by:
  • The user-specified metrics on initial nodes, if any.
  • The size to respect max_chordal_error (curved lines only) when smaller (but no smaller than min_h).
  • The local line lengths when smaller (but no smaller than min_h).
The new edge size will tend (according to the max_gradation parameter) to this value wherever possible (i.e. where not constrained by user-defined metrics, max chordal error or line lengths).
[in]metricsOptional. The user-defined mesh sizes.
The size associated to node i is defined by metrics[i].
Negative or null values are discarded.
The actual metrics may be reduced by the chordal control (max_chordal_error) if any (but no smaller than min_h).
[in,out]colorsThe line ID (color) of the elements.
The color associated to edge i (column i in connectE) is defined by colors[i].
Upon exit, all edges of the same line (between two hard nodes) are affected with the same unique color id.
If not empty upon entry:
  • Nodes between edges with different valid (i.e. not CM2_NONE) colors are considered as hard nodes and delimit lines.
  • User can provide colors for only some elements in the initial mesh (partial user coloring).
    These colors will be propagated to all the edges of each line (only the smallest color in case of multiple user colors along the same line).
  • Additional color values (when partial user coloring) start with the next highest value in colors upon entry and increase consecutively.
[in,out]hard_nodesThe extremity nodes of the lines (i.e. hard nodes).
Nodes added to partition the lines are:
  • Non-manifold nodes (i.e. connected to only one edge or to more than two edges).
  • Nodes between two edges oriented oppositely.
  • Nodes of hard_edges.
  • Angulous nodes (i.e. nodes between two edges with angle greater than angle_tol, except when one of the two edges is shorter than min_h).
  • Nodes between two edges with different valid (i.e. not CM2_NONE) user-defined colors.
[in]hard_edgesOptional. Edges (i.e. columns IDs in connectE upon entry) that shouldn't be remeshed.
[in]force_evenFlag to force the number of created edges along each line to be even.
Useful for quadmesh_iso::mesher and quadmesh_aniso::mesher in all-quad mode.
Default = false.
[in]angle_tolMaximum angle (in degree) to delimit lines.
An angle between two adjacent initial edges greater than this value will be considered a boundary node (hard node), except when one of the edges is shorter than min_h.
Default = 20.
[in]max_chordal_errorThe maximum chordal error (i.e. distance) between the new mesh and the initial mesh.
The size of the edges is reduced (but no smaller than min_h) so that the distance between the elements and the initial lines are everywhere below the maximum chordal error:
  • If max_chordal_error > 0, the chordal error is considered as absolute (same maximum chordal error everywhere).
  • If max_chordal_error < 0, the chordal error is considered as relative with respect to the local curvature radii (approximative curvatures).
    The considered max chordal error is then equal to - max_chordal_error * curvature radius.
The chordal error control is disabled when max_chordal_error = 0 or +-DBL_MAX.
Default = -0.02 (relative value, 2% of local curvature radii).
[in]max_gradationGradation of the sizes along the lines.
A value close to 0 leads to a more progressive (smoother) variation of the sizes along the lines.
When max_gradation = 0, the sizes are linearly interpolated between the sizes at the hard nodes (which may depend on connected line lengths, user-defined metrics and local curvatures).
Default = 0.5.
[in]min_hMinimum mesh size (security).
The mesh size will not be reduced (by the max_chordal_error criterion) or by local line lengths beyond this limit.
In addition, edges shorter than this limit will not be considered for the angle_tol criterion.
Default = 0 (disabled).
[in]min_nThe minimum number of edges that should be created along each line.
Default = 1.
[in]max_nThe maximum number of edges that should be created along each line.
Default = UINT_MAX (disabled).
Warning
The orientation of the initial edges is important.
The node between two edges oriented oppositely will be considered a hard node (line delimitation).
Use meshtools1d::mesh_reorient if needed.
Note
The orientation of the edges are preserved in the new mesh.
When max_chordal_error != 0, curved lines are smoothed into cubic splines and new nodes may not be exactly positioned onto the initial edge mesh.
This preserves (or even improves) for instance the geometric quality of the meshes along circles.
User-defined hard nodes (upon entry) not present in the initial mesh are removed from hard_nodes upon exit.
The node IDs always refer to columns in the coordinate matrix pos.
Returns
Error code. Negative value in case of error.
Returned value = -k => the k-th argument had an illegal value.
Precondition
Dimension = 2 or 3 (pos.rows() = 2 or 3).
connectE.empty() or vecscal::max_value(connectE) < \p pos.cols().
hard_edges.empty() or vecscal::max_value(hard_edges) < \p connectE.cols().
target_h >= 0
angle_tol >= 0
max_gradation >= 0
min_n > 0
min_n <= max_n
