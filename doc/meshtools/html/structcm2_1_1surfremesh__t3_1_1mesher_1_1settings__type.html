◆ max_chordal_error

double cm2::surfremesh_t3::mesher::settings_type::max_chordal_error

Maximum chordal error allowed.

The mesh size is reduced locally to limit the chordal error between the mesh and the initial surface:

  • If negative, this value is relative to the local radii (for instance -0.01 => max chordal error = 1% of local radii).
  • If positive, this value is absolute (for instance 0.1 => max chordal error = 0.1).

Set to DBL_MAX or a value lesser than -1 to disable.

Note
Used along ridge lines (angulous skeleton lines).
Used also to delimit patches.
For expert users only. Should not normally be changed.

Default = -0.02 (max chordal error = 2% of the local radii).

◆ patch_angle_tolerance

double cm2::surfremesh_t3::mesher::settings_type::patch_angle_tolerance

Parameter used to delimit the patches (groups of connected faces in the initial mesh).

An angle between two adjacent faces greater than this value (in degree) will draw a limit between two patches (we call such limit a "skeleton line").
A small value tends to give numerous small patches and the final mesh will be geometrically close to the initial mesh but may be of poor quality.
A large value tends to give fewer patches but bigger. The remesher can do a better job on big patches but the final mesh may be more distant from the initial mesh (i.e. greater geometric error).

A value of 45 should be considered as the maximum for this parameter.

Default = 20.

Precondition
patch_angle_tolerance >= 0.
patch_angle_tolerance <= 180.

◆ fix_tolerance

double cm2::surfremesh_t3::mesher::settings_type::fix_tolerance

Controls the merging of nodes and the swapping of edges in the initial clean-up step.

This parameter is the maximum allowable distance (absolute or relative to min_h) that a node merging or an edge swapping can induce on the initial surface.

If fix_tolerance > 0, fix_tolerance is taken as an absolute tolerance.

If fix_tolerance < 0, this is a relative tolerance.
The distance of merging is then computed as the product of -fix_tolerance and min_h, or if min_h is null, as the product of -fix_tolerance and a mean of the lengths of the edges in the initial mesh.

Note
Merging nodes in the initial mesh may lead to elements flips and intersections that could remain in the final mesh.
If this happens, reduce the magnitude of fix_tolerance.
A value of 1% of min_h (-0.01) should be considered as the maximum for this parameter.
Set this parameter to 0 to avoid any node merging/edge swapping in the initial mesh (or set initial_cleanup_flag = false).
For expert users only. Should not normally be changed.

Default = -0.0025 (0.25% of min_h).

◆ strain_tolerance

double cm2::surfremesh_t3::mesher::settings_type::strain_tolerance

The patches will be defined in such a way that when unfolded to the Z=0 plane they are not strained more than this parameter.

Set strain_tolerance = 0, if no strain should be allowed.
Only perfectly unfoldable patches will be selected (i.e. flat or simply curved patches).
The strain is computed for each element as the relative difference between the shape quality before and after unfolding.

A value of 0.60 should be considered as the maximum for this parameter.

◆ closed_manifold_flag

bool cm2::surfremesh_t3::mesher::settings_type::closed_manifold_flag

Flag to tell whether the 3-D surface should be considered as simple (manifold) and closed (watertight) or not.

In a closed manifold 3-D surface mesh the following Euler formula holds:

3*T - 2*E = 0

with:

  • T = number of triangles
  • E = number of edges

If true, a specific algorithm is used to correct the mesh when some elements are under-connected or over-connected (edges not shared exactly by two elements).

If true, the remesher emits a warning (data_type::CM2_NON_CLOSED_MANIFOLD_WARNING) if it cannot enforce all edges to be shared by exactly two elements.

Warning
This flag should not be set when intern surfaces are embedded in a solid for instance, or when notches must be kept open.
Note
Discarded when initial_cleanup_flag = false.

Default = false.

◆ line_proximity_detection_flag

bool cm2::surfremesh_t3::mesher::settings_type::line_proximity_detection_flag

Flag to take into account (true) or not (false) the distance between skeleton lines for the size of the elements.

If true, the size of the elements along close skeleton lines is reduced (depending on the distance between these skeleton lines, min_h, max_gradation_1d, max_gradation_2d).

If false, the distance between skeleton lines isn't considered for the sizes, but their lengths remain considered near their extremities (and these mesh sizes can be propagated along the skeleton lines and inside the surface patches through max_gradation_1d and max_gradation_2d).

Default = true.

◆ remesh_flag

bool cm2::surfremesh_t3::mesher::settings_type::remesh_flag

Flag to enable remeshing.

If true, remeshes the patches. In this case, the flags node_smoothing_flag, node_inserting_flag, node_removing_flag and shell_remeshing_flag are discarded (considered as true).
The coordinates of the initial nodes (all the columns of data_type::pos upon entry) are not changed.

If false, does not remesh the patches but regularizes them (depending on node_smoothing_flag, node_inserting_flag, node_removing_flag and shell_remeshing_flag).
The coordinates of the initial nodes may be changed (when node_smoothing_flag = true).

Warning
Initial clean-up and final optimization may also modify the mesh.
Disables these steps with initial_cleanup_flag = final_optimization_flag = false if needed.
Note
To get only the skeleton edges and do not change anything in the mesh, set:

◆ force_deep_analyzing_flag

bool cm2::surfremesh_t3::mesher::settings_type::force_deep_analyzing_flag

Flag to force deep analyzing of the model.

If set to false, the patch-finding algorithm starts in 'fast' mode: it assumes first that the flat patches have no self-intersection and check the meshability for curved patches only.
In case of remeshing failure (when a flat patch has self-intersection), it falls back to the deep-analysing mode (where all patches are systematically checked for non self-intersection and meshability).

If you know that your model has flat self-intersections, you can save some CPU time by setting this flag to true from the beginning.

Note
In patch-finding-only mode (remesh_flag = false, optim_level = 0) where no remeshing is performed, the algorithm cannot fall back to deep-analyzing mode.
Hence in case of flat self-intersections, correct patch-finding requires force_deep_analyzing_flag = true.

Default = false.

◆ structured_pattern

int cm2::surfremesh_t3::mesher::settings_type::structured_pattern

Meshing mode on rectangular-like patches (unstructured or structured meshes).

Two types of structured meshes: 

   - Simply-oriented pattern:

         *------*------* 
         | \    | \    |
         |   \  |   \  |
         |     \|     \|
         *------*------*
         | \    |\     |
         |   \  |  \   |
         |     \|    \ |
         *------*------*


   - "Union-Jack" pattern:        

         *------*------* 
         | \    |    / |
         |   \  |  /   |
         |     \|/     |
         *------*------*
         |     /|\     |
         |   /  |  \   |
         | /    |    \ |
         *------*------*

Possible values are:

  • -1 : Always use the frontal algorithm (generates unstructured meshes).
  • 0 : When possible, generate structured simply-oriented meshes.
  • 1 : Same as 0.
  • >= 2 : When possible, generate structured UJ meshes ("Union-Jack" pattern).
Note
Structured meshes (i.e. grid) can be generated only on rectangular-like patches and when the discretizations of the four skeleton lines match (same number of edges along opposite skeleton lines).

Default = -1 (always unstructured mesh).

◆ optim_level

unsigned cm2::surfremesh_t3::mesher::settings_type::optim_level

Optimization level between 0 and 10.

Controls the trade-off between speed (low values) and elements quality (high values).
The higher the level, the more aggressive the algorithms.

Note
The algorithms involved here aim at optimizing both the shape quality of the elements and their size quality (length of elements' edges).
Hence, increasing the value of optim_level may lead sometimes to a reduction of shape qualities (min, average or max value) whereas the global quality (which takes into account both types of quality) gets better.
The shape_quality_weight parameter controls the weight of the shape quality in the global quality measure.
Level 0 disables any optimization.

Default = 3

◆ length_upper_bound

double cm2::surfremesh_t3::mesher::settings_type::length_upper_bound

Upper bound value for the edge lengths in the final mesh.

The mesher tends to generate meshes with no edge longer than this value (normalized length).
But this is not a strict enforcement. Most of the edges will be shorter than this limit, but some may remain somewhat longer.

The default value (1.414) gives the optimal meshes with respect to the size qualities.
With this default value, the average edge length tends to be 1.

Sometimes it can be useful to limit the length of the edges to a shorter value (usually between 1 and 1.414), and to accept a mean value smaller than 1 (sub-optimal edge qualities).

Note
For expert users only. Should not normally be changed.

Default = 1.414

Precondition
length_upper_bound > 0.

unsigned cm2::surfremesh_t3::mesher::settings_type::num_threads

The number of threads for parallel processing.

The special value zero (the default) is equivalent to the number of logical processors on the running CPU.

If num_threads = 1, the mesh generator runs on a single thread.

Values are capped by the number of logical processors on the running CPU.

Default: 0 (= the number of logical processors on the running CPU).
