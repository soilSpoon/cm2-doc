Computes differences, intersection and union between (usually) two solids defined by their boundary faces (3-node triangles closed meshes only, oriented consistently).

Template Parameters
IntersectorT33-D surface triangle intersections repairer class (usually cm2::intersect_t3::mesher).
Mesher3D3-D mesh generator class (usually cm2::tetramesh_iso::mesher or cm2::tetramesh_aniso::mesher).
Parameters
[in]intersectorThe surface triangle intersections fixer (template parameter).
Repairs the surfaces in order to have valid, conformal triangle surface meshes.
[in]mesher3dThe tetrahedron mesher (template parameter).
Analyses solid topology to find subdomains and tags them as differences, intersection and union.
[in,out]dataThe surface triangle data.
Upon entry, parts (usually two) must be defined in data.xparts (past-the-end indices of the parts in data.connectM).
Upon exit, the surface of part #0 minus part #1 (difference) is returned between column #0 and column data.xparts[0] (past-the-end) of data.connectM.
The surface of part #1 minus part #0 (difference) is returned between column data.xparts[0] and column data.xparts[1] (past-the-end) of data.connectM.
The surface of the intersection is returned between column data.xparts[2] and past-last column (data.xparts[3]) of data.connectM.
In case of a single surface (i.e. data.xparts.size() = 1), part #1 is considered empty and data.xparts[1] = data.xparts[0] (difference part#1 - part #0 is empty), data.xparts[2] = data.xparts[1] (intersection is empty) and connectMU = data.connectM (union = part #0).
Parts above 2 are removed from part #0, part #1 and from intersection, but not from union (connectMU).
And no part i minus other parts is returned for them.
[out]connectMUThe surface connectivity matrix of the union of the domains (without faces common to the two parts and without any internal faces).
Specific connectivity matrix not returned in data.connectM.
Returns
Error code. Negative value in case of error.
Note
Extern surfaces should be closed (or with stitches/gaps smaller than intersector.settings.tolerance to let intersector repair them).
Extern surfaces should be oriented consistently: all inwards (all normals inward the solids) or all outwards (all normals outward the solids).
The solids shouldn't have cavities (intern closed surfaces oriented inward) within the intersection subdomain.
In case of a single surface, this function repairs any self intersection and returns the repaired outer surface (removing any intern face).
The intersector usually deteriorates locally the shape quality of the triangles (where intersections occur).
For FEA computations, a remeshing of the surfaces after this function is usually required (with cm2::surfremesh_t3::mesher or cm2::surfremesh_q4::mesher) to improve the quality of the meshes.
Upon exit we have: domain[union] = domain[part #0 - part #1] + domain[part #1 - part #0] + domain[intersection].
The time complexity of this function is roughly linear in term of the total number of nodes (O(N0 + N1)).
Warning
Internal faces in each solid are lost, except fully-closed cavities that aren't within the intersection subdomain.
Possible cases of failure:
  • Extern surfaces are not closed/watertight and intersector.settings.tolerance is smaller than the stitches/gaps.
    The function continues silently (without any error or warning) but with possible wrong results.
    Increase intersector.settings.tolerance to let the intersector repair them.
  • Extern surfaces are not all oriented consistently.
    The function continues silently (without any error or warning) but with possible wrong results.
    Reorient them (with possibly cm2::meshtools2d::mesh_reorient or with a first run of mesher3d with settings.subdomains_forcing = +1 followed by cm2::meshtools::get_mesh_boundaries).
  • Presence of cavities within the intersection subdomain.
    The function continues silently (without any error or warning) but with wrong results.
  • intersector fails to repair all intersections (can happen on almost coplanar surfaces with very sharp triangles).
    A warning is raised in data.warning_code (cm2::intersect_t3::mesher::data_type::CM2_REMAINING_INTERSECTIONS) but the function continues (with possible wrong results).
    Try to improve the quality of the initial mesh with cm2::surfremesh_t3::mesher.
  • mesher3d fails to triangulate the solids respecting the boundary meshes (in very complex cases).
    This aborts the function and returns an error code (usually cm2::tetramesh_iso::mesher::data_type::CM2_FACE_ERROR).
    Try to improve the quality of the initial mesh with cm2::surfremesh_t3::mesher.
  • Insufficient memory (huge meshes only).
    Returns cm2::intersect_t3::mesher::data_type::CM2_SYSTEM_MEMORY_ERROR (i.e. value -199).

Converts CM2_TETRA4, CM2_PYRAMID5, CM2_WEDGE6 and CM2_HEXA8 elements into higher degree elements (such as CM2_TETRA10, CM2_PYRAMID14, CM2_WEDGE18, CM2_HEXA20 and over), creating new high-order nodes.

For tetrahedrons, nodes are locally numbered as follow: 

   0                 -> 3                     : Initial summits
   4                 -> 4 +   Ne - 1          : New nodes on edge 1-2, equally spaced, from N1 to N2.
   4 + Ne            -> 4 + 2*Ne - 1          : New nodes on edge 2-0, equally spaced, from N2 to N0.
   4 + 2*Ne          -> 4 + 3*Ne - 1          : New nodes on edge 0-1, equally spaced, from N0 to N1.
   4 + 3*Ne          -> 4 + 4*Ne - 1          : New nodes on edge 3-0, equally spaced, from N3 to N0.
   4 + 4*Ne          -> 4 + 5*Ne - 1          : New nodes on edge 3-1, equally spaced, from N3 to N1.
   4 + 5*Ne          -> 4 + 6*Ne - 1          : New nodes on edge 3-2, equally spaced, from N3 to N2.
   4 + 6*Ne          -> 4 + 6*Ne + Nft - 1    : New nodes on face 1-2-3, equally spaced, row by row, left to right.
   4 + 6*Ne + Nft    -> 4 + 6*Ne + 2*Nft - 1  : New nodes on face 2-0-3, equally spaced, row by row, left to right.
   4 + 6*Ne + 2*Nft  -> 4 + 6*Ne + 3*Nft - 1  : New nodes on face 1-3-0, equally spaced, row by row, left to right.
   4 + 6*Ne + 3*Nft  -> 4 + 6*Ne + 4*Nft - 1  : New nodes on face 2-1-0, equally spaced, row by row, left to right.
   4 + 6*Ne          -> 4 + 6*Ne + Ni         : New nodes inside, equally spaced, the node close to N0 first,
                                                then the node close to N1, N2, then N3 (for the case Ni = 4).

For pyramids, nodes are locally numbered as follow: 

   0        -> 4              : Initial summits.
   5        -> 5 +   Ne - 1   : New nodes on edge 0-4, equally spaced, from N0 to N4.
   5 + Ne   -> 5 + 2*Ne - 1   : New nodes on edge 1-4, equally spaced, from N1 to N4.
   5 + 2*Ne -> 5 + 3*Ne - 1   : New nodes on edge 2-4, equally spaced, from N2 to N4.
   5 + 3*Ne -> 5 + 4*Ne - 1   : New nodes on edge 3-4, equally spaced, from N3 to N4.
   5 + 4*Ne -> 5 + 5*Ne - 1   : New nodes on edge 0-1, equally spaced, from N0 to N1.
   5 + 5*Ne -> 5 + 6*Ne - 1   : New nodes on edge 1-2, equally spaced, from N1 to N2.
   5 + 6*Ne -> 5 + 7*Ne - 1   : New nodes on edge 2-3, equally spaced, from N2 to N3.
   5 + 7*Ne -> 5 + 8*Ne - 1   : New nodes on edge 3-0, equally spaced, from N3 to N0.

   5 + 8*Ne          -> 5 + 8*Ne + Nft - 1    : New nodes on face 0-4-1, equally spaced.
   5 + 8*Ne +  Nft   -> 5 + 8*Ne + 2*Nft - 1  : New nodes on face 1-4-2, equally spaced.
   5 + 8*Ne +  2*Nft -> 5 + 8*Ne + 3*Nft - 1  : New nodes on face 2-4-3, equally spaced.
   5 + 8*Ne +  3*Nft -> 5 + 8*Ne + 4*Nft - 1  : New nodes on face 3-4-0, equally spaced.

   5 + 8*Ne + 4*Nft       -> 5 + 8*Ne + 4*Nft + Nfq - 1    : New nodes on face 0-1-2-3, equally spaced, row by row, left to right.

   5 + 8*Ne + 4*Nft + Nfq -> 5 + 8*Ne + 4*Nft + Nfq + Ni   : New node inside (generated along the segment starting from 
                                                             the center of the quad face to the summit of the pyramid).

For wedges, nodes are locally numbered as follow: 

   0        -> 5              : Initial summits.
   6        -> 6 +   Ne - 1   : New nodes on edge 0-3, equally spaced, from N0 to N3.
   6 + Ne   -> 6 + 2*Ne - 1   : New nodes on edge 1-4, equally spaced, from N1 to N4.
   6 + 2*Ne -> 6 + 3*Ne - 1   : New nodes on edge 2-5, equally spaced, from N2 to N5.
   6 + 3*Ne -> 6 + 4*Ne - 1   : New nodes on edge 1-2, equally spaced, from N1 to N2.
   6 + 4*Ne -> 6 + 5*Ne - 1   : New nodes on edge 2-0, equally spaced, from N2 to N0.
   6 + 5*Ne -> 6 + 6*Ne - 1   : New nodes on edge 0-1, equally spaced, from N0 to N1.
   6 + 6*Ne -> 6 + 7*Ne - 1   : New nodes on edge 4-5, equally spaced, from N4 to N5.
   6 + 7*Ne -> 6 + 8*Ne - 1   : New nodes on edge 5-3, equally spaced, from N5 to N3.
   6 + 8*Ne -> 6 + 9*Ne - 1   : New nodes on edge 3-4, equally spaced, from N3 to N4.

   6 + 9*Ne         -> 6 + 9*Ne +   Nft - 1  : New nodes on face 0-1-2, equally spaced.
   6 + 9*Ne +  Nft  -> 6 + 9*Ne + 2*Nft - 1  : New nodes on face 3-5-4, equally spaced.

   6 + 9*Ne + 2*Nft         -> 6 + 9*Ne + 2*Nft + Nfq - 1    : New nodes on face 2-1-4-5, equally spaced, row by row, left to right.
   6 + 9*Ne + 2*Nft + Nfq   -> 6 + 9*Ne + 2*Nft + 2*Nfq - 1  : New nodes on face 5-3-0-2, equally spaced, row by row, left to right.
   6 + 9*Ne + 2*Nft + 2*Nfq -> 6 + 9*Ne + 2*Nft + 3*Nfq - 1  : New nodes on face 0-3-4-1, equally spaced, row by row, left to right.

   6 + 9*Ne + 2*Nft + 3*Nfq -> 6 + 9*Ne + 2*Nft + 3*Nfq + Ni : New node inside (0 or 1).

For hexahedrons, nodes are locally numbered as follow: 

   0        -> 7               : Initial summits
   8        -> 8 +    Ne - 1   : New nodes on edge 0-1, equally spaced, from N0 to N4.
   8 + Ne   -> 8 +  2*Ne - 1   : New nodes on edge 1-2, equally spaced, from N1 to N5.
   8 + 2*Ne -> 8 +  3*Ne - 1   : New nodes on edge 2-3, equally spaced, from N2 to N6.
   8 + 3*Ne -> 8 +  4*Ne - 1   : New nodes on edge 3-0, equally spaced, from N3 to N7.
   8 + 4*Ne -> 8 +  5*Ne - 1   : New nodes on edge 4-5, equally spaced, from N0 to N3.
   8 + 5*Ne -> 8 +  6*Ne - 1   : New nodes on edge 5-6, equally spaced, from N4 to N7.
   8 + 6*Ne -> 8 +  7*Ne - 1   : New nodes on edge 6-7, equally spaced, from N5 to N6.
   8 + 7*Ne -> 8 +  8*Ne - 1   : New nodes on edge 7-4, equally spaced, from N1 to N2.
   8 + 8*Ne -> 8 +  9*Ne - 1   : New nodes on edge 0-4, equally spaced, from N0 to N1.
   8 + 9*Ne -> 8 + 10*Ne - 1   : New nodes on edge 1-5, equally spaced, from N3 to N2.

  • A perfect pyramid have four of its dihedral angles equal to 1.9106 (109.47 deg.), the other four equal to 0.9553 (54.74 deg).
Parameters
[in]posThe coordinate matrix (3-D).
[in]connectMThe connectivity matrix of the solid mesh (pure tetra, wedge, pyramid, hex or mixed).
[in]normalizedFlag to normalize the angles. 1 becomes the value for a perfect dihedral angle.
[out]min_anglesmin_angles[i] is the minimum dihedral angle in element i.
[out]max_anglesmax_angles[i] is the maximum dihedral angle in element i.
Returns
Error code. Negative value in case of error.
Returned value = -k => the k-th argument had an illegal value.
Precondition
pos.rows() = 3
connectM.rows() = 4 (pure tetrahedral mesh), 5 (pure pyramid mesh), 6 (pure wedge mesh), 8 (pure hex mesh or mixed mesh) or empty.
All nodes IDs referenced in connectM should be lesser than pos.cols().
Note
The computed angles for each element should always be strictly positive. Negative or null values indicate degenerated elements.
