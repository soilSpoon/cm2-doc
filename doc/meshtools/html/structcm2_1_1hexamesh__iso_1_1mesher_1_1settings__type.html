◆ strict_constraints_level

unsigned cm2::hexamesh_iso::mesher::settings_type::strict_constraints_level

Enforcement strictness level for hard nodes, edges and faces.

When 0, no error is ever raised on hard entity recovery issue (only warnings).

When 1, an error is raised when the boundary mesh has intersection(s), i.e. in the following cases:

  • A hard node is intersecting another hard entity (node, edge or face).
  • A hard edge is intersecting another hard entity (edge or face).

Note that even with a valid boundary mesh, some wall quads can be missing in the final mesh (too hard to enforce) and replaced with two triangles.
In these cases, a mere data_type::CM2_FACE_Q_DISCARDED warning is raised instead.

When 2, an error is raised in the previous case plus when a valid boundary quad could not be enforced in the final mesh.

Note
When a hard node, edge or face is outside the meshed domain (dangling entity) only a data_type::CM2_NODE_DISCARDED, data_type::CM2_EDGE_DISCARDED or data_type::CM2_FACE_DISCARDED warning is issued, even when strict_constraints_level > 0.
See also
hard_quads_enforcement_flag, conformation_level.

Default = 1

◆ subdomains_forcing

int cm2::hexamesh_iso::mesher::settings_type::subdomains_forcing

Forcing mode for intern subdomains.

Possible values are:

  • -1 : All intern subdomains are considered as holes regardless of the orientation of their enclosing boundary.
  • 0 : Intern subdomains are considered as holes when the orientation of their enclosing boundary is opposite to the orientation of the most enclosing domain.
    Otherwise, they are meshed.
  • +1 : All intern subdomains are meshed regardless of the orientation of their enclosing boundary.
Note
Used only in MESH_MODE.

Default = 0.

◆ refine_flag

bool cm2::hexamesh_iso::mesher::settings_type::refine_flag

Flag to enable/disable refinement.

This flag enables/disables the generation of new points inside the domain in order to get good elements (in shape and size).

With this flag down (unset), the mesher triangulates only the domain, i.e. it stops after the front mesh step.
Only the hard nodes and possibly some Steiner nodes (see conformation_level and hard_quads_enforcement_flag) will be in the final mesh (with usually only tetrahedrons and some pyramids).

Note
Not used in CONVEX_HULL_MODE.
In REGULARIZE_MODE, unsetting this flag is equivalent to unsetting node_inserting_flag.

Default = true.

◆ use_default_background_mesh_flag

bool cm2::hexamesh_iso::mesher::settings_type::use_default_background_mesh_flag

Flag to tell the mesher to use the default background mesh for the interpolation of the metric values.

For expert users only.

The default background mesh is the "front" mesh, i.e. the mesh of the triangulation of the hard nodes without any new node.

When unset, the background mesh used to interpolate the metrics is the mesh currently processed.

Note
Using the current mesh is the fastest mode which indeed gives a similar accuracy.
It is recommended to leave this flag to false.
For expert users only.
Warning
A user-defined background mesh supersedes the default background mesh, even with this flag set.

Default = false.

◆ structured_flag

bool cm2::hexamesh_iso::mesher::settings_type::structured_flag

Flag to enable/disable structured (grid-like) meshing is some cases.

Structured mesh (grid-like) requires specific conditions:

  • The meshes of the opposite faces match (similar meshes).
  • Empty data_type::connectE (no embedded edges).
  • Empty data_type::isolated_nodes (no embedded nodes).

In addition, this flag is discarded when metrics are used. This implies:

  • No valid (i.e. strictly positive) target_metric.
  • No valid (i.e. strictly positive) data_type::metrics (this forbids the usage of background mesh and repulsive points).
Note
With this flag set, all-hex meshes can be generated in more cases (but hexes can be quite flat and edge length qualities can be bad too).
Used only when refine_flag = true.
Used in MESH_MODE only.

Default = true.

◆ conformation_level

unsigned cm2::hexamesh_iso::mesher::settings_type::conformation_level

Level of conformation in the mesh.

In an hex-dominant mesh, joints between elements can be:

  • Conformal: T3-T3 or Q4-Q4 joints (no T3-Q4 joint).
  • Non-conformal: T3-Q4 joint (a quadrangle is jointed to two triangles).

In a conformal mesh, a hexahedron for instance can only have other hexahedrons as neighbors or wedges by one of their quadrangle faces or pyramids by their base face, but no tetrahedron.
In any case, no quadrangle face can be jointed partially (by three nodes) to another quadrangle face.

The value for this setting can be:

  • 0 : Minimal conformation: nothing specific is done to force conformal joints in the mesh.
    Non-conformal joints are possible everywhere in the mesh (but many joints can be still be conformal).
  • 1 : Partial conformation: conformal joints are enforced only near the hard quadrangles.
    Joints elsewhere inside the domain can still be non-conformal.
  • 2+: Full conformation: conformal joints are enforced everywhere in the mesh.
Note
Forcing the conformation often requires adding new nodes to break hexahedrons and wedges into smaller elements (pyramids or tetrahedrons) with reduced shape and length qualities.
More nodes, more pyramids and tetrahedrons, fewer hexahedrons and wedges.
Partial conformation (conformation_level = 1) or full conformation (conformation_level >= 2) is required to recover all boundary quads (see below hard_quads_enforcement_flag).
Used in the MESH_MODE only.
See also
hard_quads_enforcement_flag.

Default = 2 (full conformation).

◆ hard_quads_enforcement_flag

bool cm2::hexamesh_iso::mesher::settings_type::hard_quads_enforcement_flag

Flag to enable/disable the recovery of hard quadrangles (as much as possible).

With the present hex-dominant meshing scheme, hard (i.e. usually boundary) quadrangles aren't naturally all present in the mesh and can appear in the solid mesh as two triangles (triangle faces of some tetrahedrons, pyramids or wedges).
Specific algorithms (local remeshings, node moving, elements merging...) are required to recover these quads (and make them the base quad face of a pyramid or a quad face of a wedge or a face of an hexahedron).

This flag enables/disables these algorithms.

Here again, the forcing of boundary quadrangles usually means more nodes, more pyramids and tetrahedrons, fewer hexahedrons and wedges.

Note
For best recovery, set also conformation_level >= 1.
On difficult meshes, some boundary quads may still be missing even when hard_quads_enforcement_flag is set and conformation_level >= 1 raising a data_type::CM2_FACE_Q_DISCARDED warning (strict_constraints_level <= 1) or a data_type::CM2_FACE_Q_ERROR (strict_constraints_level >= 2)
Used in the MESH_MODE only.
See also
conformation_level, strict_constraints_level, enforcement_quality_threshold.

Default = true.

◆ optim_level

unsigned cm2::hexamesh_iso::mesher::settings_type::optim_level

Optimization level between 0 and 10.

Controls the trade-off between speed (low values) and elements quality (high values).
The higher the level, the more aggressive the algorithms.

Note
The algorithms involved here aim at optimizing both the shape quality of the elements and their size quality (length of elements' edges).
Hence, increasing the value of optim_level may lead sometimes to a reduction of shape qualities (min, average or max value) whereas the global quality (which takes into account both types of quality) gets better.
The shape_quality_weight parameter controls the weight of the shape quality in the global quality measure.
Level 0 disables any optimization.

Default = 3

◆ enforcement_quality_threshold

double cm2::hexamesh_iso::mesher::settings_type::enforcement_quality_threshold

Threshold for the boundary enforcement scheme (wall-quad enforcement).

The boundary enforcement scheme (to force boundary quads) may generate poor elements in the vicinity of some wall quads.

Increasing the value of this parameter may decrease the number of enforced boundary quads (wall quads), but help improve the quality of the elements near these quads.

Conversely, a lower value forces more quads but can lead to lower wall element qualities.

Default = 1E-4

Warning
For expert users only.
Note
The unenforced quads will be split and replaced with two triangles in the boundary of the resulting mesh.
Not used when strict_constraints_level = 2 (equivalent to enforcement_quality_threshold = 0).
Precondition
enforcement_quality_threshold >= 0.
enforcement_quality_threshold <= 1.

◆ interrupt_hdl

interrupt_handler_type cm2::hexamesh_iso::mesher::settings_type::interrupt_hdl

The user-provided interrupt function.

When this function returns true, the process is stopped.
This can abort the mesh (if the interruption occurred at an early stage), or result in a mesh of poor quality (if the mesher was in the refinement or optimization stage when the interruption occurs).

The parameter progress (between 0 and 1) gives a hint about the progress of the meshing.
