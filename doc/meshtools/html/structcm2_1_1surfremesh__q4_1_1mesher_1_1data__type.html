◆ warning_type

enum cm2::surfremesh_q4::mesher::data_type::warning_type

Warning enums for the remesher.

Non fatal.

If such a warning is raised (returned in the field warning_code), the remesher is not stopped but the user shall check its input and output data for a potential mistake.

High-value warnings have higher precedence than low-value warnings (except CM2_NO_WARNING = 0).
This means than a CM2_SHAPE_QUALITY_WARNING won't overwrite a CM2_INTERRUPTION even if some elements have a bad quality (Qs < 0.01).

Enumerator
CM2_NO_WARNING 

OK, no warning.

CM2_INTERRUPTION 

The process has been interrupted (by the user through the interrupt handler).

CM2_NON_CLOSED_MANIFOLD_WARNING 

The 3-D surface mesh could not be made simple and closed.

This warning can be raised only if closed_manifold_flag = true.

CM2_NODES_LIMIT_WARNING 

The limitation on the number of nodes has been reached.

CM2_SHAPE_QUALITY_WARNING 

One or several elements have a bad quality (Qs < 0.01).

◆ connectM

UIntMat cm2::surfremesh_q4::mesher::data_type::connectM

The connectivity of the mesh, input and output (column oriented).

This matrix stores the connectivity of the mesh upon entry (dimension 3 x nefs_in or 4 x nefs_in) and the connectivity of the quad-dominant or all-quad mesh upon exit (dimension always 4 X nefs).

Upon exit, the leading part of this matrix (from col 0 to col nefs_Q4 - 1) is the connectivity of the quadrangle elements.
The tailing part (from col nefs_Q4 to col nefs - 1) is the connectivity of the triangle elements.
The fourth node ID in this triangle part is always CM2_NONE (i.e. unsigned(-1)).

Upon entry, triangles and quadrangles can be freely interleaved.

Note
The values are node indices and refer to columns in the coordinate matrix pos.
To keep only the nodes of the final mesh, use meshtools::simplify.
Warning
In case of failure of the remeshing (when an error such as CM2_NODES_LIMIT_ERROR, CM2_DEGENERATED_ELEMENT, CM2_REMESHING_ERROR, CM2_SYSTEM_MEMORY_ERROR or CM2_INTERNAL_ERROR is raised), this connectivity matrix is returned empty and the initial matrix is lost.
Hence, a copy of the input connectivity is recommended before the remeshing:
UIntMat connectMsav;
connectMsav.copy(data.connectM);
cm2::dense2D::copy
bool copy(const self_type &x)
Note that the coordinates of the initial nodes in matrix pos are never changed by the remesher.

Mode = IN-OUT.

Examples

◆ metrics

DoubleVec cm2::surfremesh_q4::mesher::data_type::metrics

Mesh sizes near initial nodes.

metrics[i] is the target mesh size near node i (which coordinates are at column i in matrix pos).

Note
Set settings_type::max_gradation_1d = 0, if you don't want sizes to tend to settings_type::target_h along skeleton lines but be linearly interpolated between valid metric values.
Set settings_type::max_gradation_2d = 0, if you don't want sizes to tend to settings_type::target_h inside patches but be linearly interpolated between valid metric values.
Only the values for the nodes in the initial mesh are considered.
For nodes with null mesh size (or missing), an automatic mesh size is computed using an average distance to their neighbours.
Upon exit, this vector contains at least the user-defined values plus computed values on all other skeleton nodes.
When settings_type::compute_Qh_flag = true, it is extended to all final nodes.

Mode = IN-OUT.

◆ skeleton_edges_in

UIntMat cm2::surfremesh_q4::mesher::data_type::skeleton_edges_in

Upon entry, the user can add into this matrix some specific edges to force the partitioning algorithm to consider them as boundaries of the patches (so-called skeleton edges).

Upon exit, this matrix contains all the boundaries of the patches in the initial mesh.
(specified by user or added by the partitioning algorithm).

Note
Another convenient way to provide skeleton edges is by user coloring. See colors_in below.
Upon entry, only edges of the initial mesh are taken into account.
Simply-connected skeleton edges make a so-called skeleton line.
The skeleton lines may be remeshed: the initial nodes along these lines (except the two extremities) are replaced with new ones (usually at different coordinates).
To prevent the removal of the old nodes along a skeleton line, one solution is to put their node IDs in the hard_nodes vector.
This does not prevent however new nodes from being added between them.
To go further, place the edges in the hard_edges matrix to make them unremeshable (orientation doesn't matter).
User-specified skeleton lines upon entry may be remeshed but they are guaranteed however to remain in the final mesh, i.e. they will not be crossed by new elements.

Mode = IN-OUT

◆ exclude_skeleton_edges

UIntMat cm2::surfremesh_q4::mesher::data_type::exclude_skeleton_edges

This matrix can be used to exclude some edges from being selected by the partitioning algorithm as skeleton edges, if possible.

The user can use this matrix to make the algorithm select bigger patches than it normally would.

Note
Only the edges that break solely the settings_type::patch_angle_tolerance criterion can be excluded this way.
The algorithm will keep a skeleton edge if it breaks any other criterion (settings_type::strain_tolerance or if this edge is necessary to achieve a successful remeshing).

◆ colors_in

UIntVec cm2::surfremesh_q4::mesher::data_type::colors_in

The patch ID (color) of the initial elements.

Automatic coloring, when colors_in is empty upon entry:

  • The partitioning algorithm use its own criterions (settings_type::patch_angle_tolerance, settings_type::strain_tolerance, settings_type::max_chordal_error, skeleton_edges_in, exclude_skeleton_edges) to partition the initial mesh and set colors to the patches (one color value by patch with continuous numbering starting with 0).
  • User has little control over the patches (only through the above criterions) and none over the color values.

User coloring, when colors_in is not empty and has valid color values (i.e. different from CM2_NONE) upon entry:

  • The boundaries between different valid colors are considered as skeleton edges and honored as such.
  • User can provide valid colors for only some elements in the initial mesh (partial user coloring).
    The valid user color for the element with lowest ID will be assigned to all elements of the patch this element belongs to (discarding any other user color in the same patch).
  • The partitioning algorithm can add other skeleton edges (angulous edges or in order to respect other criterions such as settings_type::strain_tolerance, settings_type::max_chordal_error...)
  • Additional color values (when partial user coloring) start with the next highest value different from CM2_NONE given by user in colors_in upon entry and increase consecutively.
Note
Element i at connectM.col(i) upon entry (initial mesh) is in the patch with color colors_in[i], like all other elements with same color.
New elements in the final mesh are colored (in array colors) with same color as their parent element in the initial mesh (whatever the type of coloring, automatic or user coloring).
Patches with different properties that should be kept over the remeshing (such as material properties, boundary conditions...) must be separated from each other upon entry by user coloring, or at least, when automatic coloring, by skeleton lines (skeleton_edges_in or hard_edges).
Elements not colored by user (automatic or partial user coloring) and eliminated during the initial clean up step are assigned with no color (CM2_NONE).
Disable the initial clean up step (set settings_type::initial_cleanup_flag = false) to avoid this.
Warning
In case of several consecutive remeshings, set colors_in = colors, skeleton_colors_in = skeleton_colors and skeleton_edges_in = skeleton_edges (or at least clear them).
Precondition
colors_in.size() <= connectM.cols() upon entry (i.e. nefs_in).
Postcondition
colors_in.size() = nefs_in.

Mode = IN-OUT
