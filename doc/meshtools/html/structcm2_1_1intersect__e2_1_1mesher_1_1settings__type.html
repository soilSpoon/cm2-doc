Detailed Description

The settings of intersect_e2::mesher.

This structure gathers flags and parameters to control the way the intersect_e2::mesher works.

typedef bool(* cm2::intersect_e2::mesher::settings_type::interrupt_handler_type) (void *pass_thru, double progress)

Typedef for the interrupt handler.

The interrupt handler, if any, is called internally by the mesher from time to time to query the abortion of process (when the handler returns true) or to continue the meshing process (when the handler returns false).

Parameters
[in]pass_thruThe settings_type::pass_thru pointer (user provided).
[in]progressValue (between 0 = just starting and 1 = meshing is complete) to give an estimate of the progress of the meshing.

◆ tolerance

double cm2::intersect_e2::mesher::settings_type::tolerance

Intersection tolerance.

Nodes closer than this value are merged.
Edges closer than this value are split and their intersection merged.

Priority rule is: lowest edges have higher priority.
Nodes and intersecting points are merged into nodes and points of the face with lowest ids in data_type::connectM.

Default = 1E-12

Examples
tmsh2di07.cpp, and tmsh2di50.cpp.

◆ analysis_code

unsigned cm2::intersect_e2::mesher::settings_type::analysis_code

Type of analysis:

  • 0 : Inter analysis. Only pathologies between elements of different parts are considered.
  • 1 : Intra analysis. Only pathologies between elements of the same part are considered.
  • 2+ : Full analysis. Pathologies between all elements are considered (discarding parts info).

Default = 0 (inter analysis).

Note
When data_type::xparts.size() <= 1 a full analysis is always performed.
See also
data_type::xparts.

bool cm2::intersect_e2::mesher::settings_type::dry_run_flag

Flag to disable (true) the fixing of pathologies.

Do not attempt to fix the pathologies when true. The pathologies are computed (reported in data_type::pathology_IDs_in) but not fixed.

◆ pass_thru

void* cm2::intersect_e2::mesher::settings_type::pass_thru

The pointer that will be passed to the display_hdl and interrupt_hdl (if any) when called by the mesher.

Example:

void my_display_hdl (void* pass_thru, unsigned level, const char* msg)
{
window_type* my_window = static_cast<window_type*>(pass_thru);
my_window->show(msg);
}
mesher my_mesher;
data_type my_mesh_data(pos, connectB);
window_type my_window; // A window instance.
my_mesher.settings.display_hdl = &my_display_hdl;
my_mesher.settings.pass_thru = static_cast<void*>(&my_window);
my_mesher.run(my_mesh_data); // Will call my_display_hdl.
