Computes differences, intersection and union between (usually) two 2-D domains defined by their boundary edges (2-node edges closed contour meshes only, oriented consistently).

Template Parameters
IntersectorE22-D edges contour intersections repairer class (usually cm2::intersect_e2::mesher).
Mesher2D2-D mesh generator class (usually cm2::triamesh_iso::mesher or cm2::triamesh_aniso::mesher).
Parameters
[in]intersectorThe contour intersections fixer (template parameter).
Repairs the contours in order to have valid, conformal edge meshes.
[in]mesher2dThe 2D mesher (template parameter).
Analyses domain topology to find subdomains and tags them as differences, intersection and union.
[in,out]dataThe contour edges data.
Upon entry, parts (usually two) must be defined in data.xparts (past-the-end indices of the parts in data.connectM).
Upon exit, the contour of part #0 minus part #1 (difference) is returned between column #0 and column data.xparts[0] (past-the-end) of data.connectM.
The contour of part #1 minus part #0 (difference) is returned between column data.xparts[0] and column data.xparts[1] (past-the-end) of data.connectM.
The contour of the intersection is returned between column data.xparts[2] and past-last column (data.xparts[3]) of data.connectM.
In case of a single contour (i.e. data.xparts.size() = 1), part #1 is considered empty and data.xparts[1] = data.xparts[0] (difference part#1 - part #0 is empty), data.xparts[2] = data.xparts[1] (intersection is empty) and connectMU = data.connectM (union = part #0).
Parts above 2 are removed from part #0, part #1 and from intersection, but not from union (connectMU).
And no part i minus other parts is returned for them.
[out]connectMUThe contour connectivity matrix of the union of the domains (without edges common to the two parts and without any internal edges).
Specific connectivity matrix not returned in data.connectM.
Returns
Error code. Negative value in case of error.

Converts CM2_FACET3 and CM2_FACEQ4 elements into high-order elements (such as CM2_FACET6, CM2_FACEQ8 and CM2_FACEQ9), creating new high-order nodes.

For triangle elements, nodes are locally numbered as follow:

  • 0 -> 2 : Initial summits
  • 3 -> 3 + Ne - 1 : New nodes on edge 1-2, equally spaced, from N1 to N2.
  • 3 + Ne -> 3 + 2*Ne - 1 : New nodes on edge 2-0, equally spaced, from N2 to N0.
  • 3 + 2*Ne -> 3 + 3*Ne - 1 : New nodes on edge 0-1, equally spaced, from N0 to N1.
  • 3 + 3*Ne -> 3 + 3*Ne + Ni : New nodes inside. Equally spaced, numbered row by row, left to right, starting with the node close to node 2.

For quadrangles, nodes are locally numbered as follow:

  • 0 -> 3 : Initial summits
  • 4 -> 4 + Ne - 1 : New nodes on edge 0-1, equally spaced, from N0 to N1.
  • 4 + Ne -> 4 + 2*Ne - 1 : New nodes on edge 1-2, equally spaced, from N1 to N2.
  • 4 + 2*Ne -> 4 + 3*Ne - 1 : New nodes on edge 2-3, equally spaced, from N2 to N3.
  • 4 + 3*Ne -> 4 + 4*Ne - 1 : New nodes on edge 3-0, equally spaced, from N3 to N0.
  • 4 + 4*Ne -> 4 + 4*Ne + Ni : New nodes inside. Equally spaced, numbered row by row, left to right, starting with the node close to node 0.

Interpolates a scalar field (doubles) defined on the nodes of a plane mesh (all-triangle, all-quad or mixed, linear or quadratic).

The interpolated values are computed on a specific set of nodes that are not in the mesh.

Parameters
[in]posThe 2-D coordinate matrix (2xNODS).
[in]connectMThe connectivity matrix of the mesh (3xNEFS, 4xNEFS, 6xNEFS, 8xNEFS or 9xNEFS).
Each node of the mesh should have a valid associated field value.
[in]neighborsThe neighbors matrix (3xNEFS).
[in]ancestorsThe ancestors vector. For each node referenced in connectM, the vector gives one face connected to it.
For nodes not in connectM, the ancestor should be CM2_NONE (i.e. unsigned(-1)).
[in,out]fieldThe scalar field (vector of size NODS).
[in]nodesThe node IDs for which the field values must be computed.
Values for nodes of the mesh are not recomputed.
[in]FE_typeThe type of the elements (CM2_FACET3, CM2_FACEQ4, CM2_FACE_MIX).
Returns
Error code. Negative value in case of error.
Returned value = -k => the k-th argument had an illegal value.
Note
The neighbors matrix is provided by every CM2 mesher or can be computed by meshtools::get_neighbors.
The ancestors vector is provided by every CM2 mesher or can be computed by meshtools::get_ancestors.
Precondition
pos.rows() == 2.
connectM.rows() = 3 or 6 (i.e. triangle mesh), connectM.rows() = 4, 8 or 9 (all-quad or mixed mesh) or empty.

Generates a structured CM2_FACEQ4 mesh (regular grid).

Takes boundary points as input and creates new points inside and returns element connectivity as output. The boundary points define a closed and preferably convex domain.

Example: 

Mesh of a square

INPUT:

   pos = [0  1  2  2  2  1  0  0
          0  0  0  1  2  2  2  1]   (stored column-wise)

   indices = [0 1 2 3 4 5 6 7 0]

   N1    = 2


OUTPUT:
                                   __ new node
                                  |
                                  V

   pos = [0  1  2  2  2  1  0  0  1
          0  0  0  1  2  2  2  1  1]      (stored column-wise)

   connectM  = [0 7 1 8                   (stored column-wise)
                1 8 2 3
                8 5 3 4                
                7 6 8 5]                  (element are always oriented as the external contour) 

   4 elements:

            
                 line #2
            6      5
            *------*------*4      ^ 
            |      |      |       |
            |      |      |       |
line #3     |     8|      |       |
N1 elems   7*------*------*3      | line #1 : N1 elements
            |      |      |       |
            |      |      |       |
            |      |      |       |
            *------*------*       |
            0      1      2
                 line #0
Parameters
[in,out]posThe coordinate matrix. New points are appended.
[in]indicesThe indices of the boundary nodes, successively, with no duplicate except for the first and the last ones, which should be equal to define a closed contour.
[in]N1The number of elements along line #1 (= number of elements along line #3).
[in,out]connectMThe connectivity of the new faces are appended.
Returns
The number of created faces (normally N0 * N1). Negative value in case of error.
Returned value = -k => the k-th argument had an illegal value.
Precondition
Number of element along line #0 = number of elements along line #2. This implies that indices.size()-1 - 2*N1 be even. N0 = N2 = (indices.size()-1 - 2*N1) / 2
pos.rows() = 2 or 3 upon entry.
connectM.rows() = 3 or 4 or empty upon entry.
All the values in indices are smaller than pos.cols().
The first index must be duplicated at the end of indices: indices().front() == indices.back().
Postcondition
(N0-1)*(N1-1) new points are appended to pos.
N0*N1 new faces are appended to connectM.
Note
The mesh is generated onto the Coons surface defined by the contour lines.
The quality of the generated elements is not checked (elements may be warped because of bad contour).

Generates a pseudo-structured mesh on a disk quarter (upper-right quarter) with CM2_FACEQ4 elements.

The mesh is made of three structured "patches": 

        6* -------____ 
         |             - *5
         |             /   \
         |            /      |
         |           /        |
        3*----------*2        |
         |          |          |
         |          |          |
         |          |          |
         |          |          |
         *----------*----------* 
         0          1          4
Parameters
[in,out]posThe coordinate matrix. New points are appended.
Points 0 to 6, then contour points, then inner surface points.
[in]CThe centre of the disk.
[in]RThe radius of the disk.
[in]NThe number of elements along each patch (=> 2*N along horizontal, vertical and arc lines).
[in,out]connectMThe connectivity of the new faces are appended.
[in,out]connectEThe connectivity of the boundary edges are appended.
Returns
The number of created faces (normally 3 * N * N). Negative value in case of error.
Returned value = -k => the k-th argument had an illegal value.
Precondition
pos.rows() = 2 or empty upon entry.
connectM.rows() = 3 or 4 or empty upon entry.
connectE.rows() = 2 or empty upon entry.
Postcondition
3*N*(N+1)+1 new points are appended to pos.
3*N*N new faces are appended to connectM.
6*N new edges are appended to connectE.
Note
The node at centre C is the first point appended to pos.
The new faces are oriented normal up.
The new edges are oriented counter clock-wise starting from arc node 4 (the rightmost node, i.e. the X-most node), then node 5 (mid arc node), then node 6 (top Y-most node), then node 3 (mid vertical node), then node 0 (centre node at C), then node 1 (mid horizontal node) and back to node 4.
The mesh is symmetrical with respect to the diagonal 45 line.

Meshes a parametric 3-D surface using a 2-D anisotropic mesher and mappings.

This function is a template function to allow the user to mesh her/his own parametric surface.

Another advantage of the template function is that an experienced user can freely adapt it to his specific needs.

The meshes generated can be isotropic on the surface (i.e. triangles tend to be equilateral and quadrangles tend to be square) or anisotropic on the surface depending on the 3-D metrics provided by the user (or if anisotropic chordal control is specified).

A default metric field is computed to produce isotropic meshes on the surface.

The user can also do adaptive meshes (isotropic or anisotropic) on the surface with the help of a "background mesh" (field data3d.background_mesh) and a set of metrics at each node of this mesh (field data3d.metrics). The background mesh is also used by this function to compute the local bases on the surface.

Template Parameters

The Surface template class should be equipped with the following members:

int get_tangents (const DoubleMat& pos2d, DoubleMat& T);
int get_curvatures (const DoubleMat& pos2d, DoubleMat& C);

The Surface::get_3D_coordinates function should compute the 3-D coordinates of a set of 2-D points located on the reference plane (UV).
The 3-D coordinates associated to the point in column j of pos2d must be returned in column j of pos3d.
This function should return zero when successful and a negative value (-1 for instance) when failed.

The Surface::get_2D_coordinates function is the reciprocal function of the previous one.
It should give the 2-D reference coordinates (UV) of a set of 3-D points.
The reference coordinates associated to the point at column j of pos3d must be returned in column j of pos2d.
The parameter nodes can be used for fast retrieving of the UV coordinates when they have been pre-computed (experts only).
If used, the pre-computed UV coordinates (stored by user in the Surface object) for node id = nodes[j] is copied straight to column j of pos2d without using its 3-D coordinates (column j of pos3d).
Note that this function is called only for the hard nodes (nodes of the boundary mesh, the background mesh and internal hard nodes, if any) or for high-order nodes when high_order_type > 0.
This function should return zero when successful and a negative value (-1 for instance) when failed.

The Surface::get_tangents function computes the two tangents Tu and Tv on the surface at a set of points given by their reference coordinates. These tangents must not be normalized.
They are the mere derivatives of the surface with respect to the two reference parameters.
The two tangents at the point in column j of pos2d must be returned in column j of T.
The first three values are for the first tangent (with respect to the first ref coordinate), then the next three are for the second tangent.
This function should return zero when successful and a negative value (-1 for instance) when failed.

The Surface::get_curvatures function computes the curvatures on the surface at a set of points given by their reference coordinates.
The curvatures C are 2x2 symmetric matrices defined as:

  • C(0,0) = <DTu/Du, N> Dot product between the derivative of Tu (first local tangent) with respect to u (i.e. second derivative of P(u,v) with respect to u, u) and the normal N to the surface.
  • C(0,1) = <DTu/Dv, N> Dot product between the derivative of Tu (first local tangent) with respect to v, or Bv (second local tangent) with respect to u (i.e. second derivative of P(u,v) with respect to u, v) and the normal N to the surface.
  • C(1,1) = <DTv/Dv, N> Dot product between the derivative of Tv (second local tangent) with respect to v (i.e. second derivative of P(u,v) with respect to v, v) and the normal N to the surface. These three values should be stored column-wise in matrix curvatures.
    This function should return zero when successful and a negative value (-1 for instance) when failed.
    Should you leave the implementation of this member empty (returning -1 for instance), approximative curvatures computed from the variations of the tangents will be used instead.

It is the responsibility of the user to provide this template function with a coherent Surface class.
That is, a class for which get_2D_coordinates and get_3D_coordinates do reciprocal computations, and get_tangents computes the derivatives of the surface with respect to the two reference parameters, and get_curvatures computes the H matrix as defined above.
This template function does not control this coherence.

Should the parametric surface be not regular on the domain to be meshed (i.e. with some singularity or degeneracies leading to null or collinear tangent vectors) this template functions will use default data instead.
This may lead however to locally poor mesh quality (the mesh can then be remeshed by CM2 SurfRemesh T3/Q4).

"Similar orientation" for elements means that each edge is oriented oppositely in the two adjacent elements.

The algorithm involved is a neighboring search starting from element ref_element.

If code == 0, only the elements in the same subdomain(1) as element ref_element will be visited. Other elements will remain unchanged.
If code != 0, all elements will be visited. Elements in other subdomains than that of ref_element, if any, will be reoriented as the first (lowest ID) element in each subdomain.

Note
