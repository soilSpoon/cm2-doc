◆ min_h

double cm2::surfmesh_t3::mesher::settings_type::min_h

The minimum mesh size (security).

The mesh size will not be reduced (by max_chordal_error or max_chordal_error_ratio) below this value:

  • If negative, this value is relative to target_h (min size = - min_h * target_h).
  • If positive, this value is absolute (min size = target_h).

Default = -1E-3 (1E-3 * target_h).

◆ fix_tolerance

double cm2::surfmesh_t3::mesher::settings_type::fix_tolerance

The merge tolerance for vertices and curves.

CAD entities are merged when their maximum distance is below or equal to the merge tolerance:

  • If negative, this value is relative to target_h (merge tolerance = - fix_tolerance * target_h).
  • If positive, this value is absolute (merge tolerance = fix_tolerance).

Vertices within the merge tolerance are merged. Curves approximately within the merge tolerance are merged.

Note
Considered for merging only vertices and curves (not for surfaces so far).
Set to +- DBL_MAX to completely disable the merging.

Default = -1E-4.

◆ max_chordal_error

double cm2::surfmesh_t3::mesher::settings_type::max_chordal_error

The maximum chordal error allowed between exact curves/surfaces and linear elements.

The mesh size will be reduced locally to limit the chordal error between the mesh and the curves or the surfaces:

  • If negative, this value is relative to the local radii (for instance -0.02 => max chordal error = 2% of local radii).
  • If positive, this value is absolute (for instance 0.1 => max chordal error = 0.1).

A complete circle is meshed with approximately:

  • 10 linear elements for max_chordal_error = -0.05 (5% of radius).
  • 13 linear elements for max_chordal_error = -0.03 (3% of radius).
  • 16 linear elements for max_chordal_error = -0.02 (2% of radius).
  • 22 linear elements for max_chordal_error = -0.01 (1% of radius).
Note
The considered chordal error is between the curves/surfaces and the linear elements.
In case of quadratic elements (high_order_type >= 1 and high_order_mode >= 1), the actual chordal error is well below.
With linear elements of size h along a curve of radius R, the chordal error is approximately given by .
The chordal error is divided by 4 when the size h is divided by 2.
With quadratic elements of size h along a curve of radius R, the chordal error (or rather the maximum distance between the quadratic element and the curve) is approximately given by .
The chordal error is divided by 16 when the size h is divided by 2.

Default = -0.02 (max error = 2% of the local radii).

◆ max_chordal_error_ratio

double cm2::surfmesh_t3::mesher::settings_type::max_chordal_error_ratio

The maximum ratio between chordal errors.

The size of the mesh along the smallest curvature direction (largest radius) is reduced (but not below min_h) so that the chordal error remains below max_chordal_error_ratio * size of the mesh along highest curvature (smallest radius).

This parameter limits the anisotropy of the elements.

It is used only with anisotropic control (chordal_control_type = 2 or 4) and when the surface has double curvature.

Note
max_chordal_error_ratio > 0.

Default: 0.10

◆ chordal_control_type

unsigned cm2::surfmesh_t3::mesher::settings_type::chordal_control_type

The chordal control type:

  • 0 : No curvature is computed and the chordal error control is disabled.
  • 1 : Isotropic chordal control based on approximate curvature (least-square method using the local bases).
    Element sizes are limited by the same (isotropic) chordal value.
  • 2 : Anisotropic chordal control based on approximate curvature (least-square method using the local bases).
    Element sizes are limited by two different (anisotropic) chordal values along the two principal curvature directions.
  • 3 : Isotropic chordal control based on exact curvature (provided by the surface curvatures).
    Element sizes are limited by the same (isotropic) chordal value.
  • 4+: Anisotropic chordal control based on exact curvature (provided by the surface curvatures).
    Element sizes are limited by two different (anisotropic) chordal values along the two principal curvature directions.

Default = 4

The gradation of the elements size.

This parameter controls the gradation of the elements size along the skeleton lines and inside the patches towards the target_h mesh size.

A value close to 0 leads to a more progressive variation of mesh size (smoother).

  • 1+: Convert to quadratic elements (CM2_FACET6).

◆ high_order_mode

unsigned cm2::surfmesh_t3::mesher::settings_type::high_order_mode

The method for positioning of the high-order nodes (HON):

  • 0 : The HON are interpolated between the linear 3D nodes. Elements remain straight.
    This method is the fastest.
  • 1 : The HON are on the curves/surfaces and have reference coordinates in the middle between the linear nodes but may not be equidistant between the linear 3D nodes. Elements are curved.
    The speed of this method is between methods 0 and 2.
  • 2+: The HON are on the curves/surfaces and equidistant between the linear 3D nodes but may not have reference values in the middle between the linear nodes. Elements are curved.
    This method is the slowest.

Relevant only when high_order_type > 0.
Default: 2.

◆ num_threads

unsigned cm2::surfmesh_t3::mesher::settings_type::num_threads

The number of threads for parallel processing.

The special value zero (the default) is equivalent to the number of logical processors on the running CPU.

If num_threads = 1, the mesh generator runs on a single thread.

Value is capped by the number of logical processors on the running CPU.

Default: 0 (= the number of logical processors on the running CPU).

void* cm2::surfmesh_t3::mesher::settings_type::pass_thru
