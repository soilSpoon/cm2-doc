Detailed Description

The data type for triamesh_aniso::mesher.

This structure is used to gather all the input and output data of the function triamesh_aniso::mesher_anisotropic::run.

Examples
tmsh2da51.cpp, and tmsh2da56.cpp.

◆ warning_type

enum cm2::triamesh_aniso::mesher::data_type::warning_type

Warning enums for the mesher.

Non fatal.

If such a warning is raised (returned in the field warning_code), the mesher is not stopped but the user shall check its input and output data for a potential mistake.

High-value warnings have higher precedence than low-value warnings (except CM2_NO_WARNING = 0).
This means than a CM2_SHAPE_QUALITY_WARNING won't overwrite a CM2_BOUNDARY_WARNING even if some elements have a bad quality (Qs < 0.01).

Enumerator
CM2_NO_WARNING 

OK, no warning.

CM2_INTERRUPTION 

The process has been interrupted (by the user through the interrupt handler).

CM2_NODES_LIMIT_WARNING 

The limitation on the number of nodes has been reached.

CM2_NODE_DISCARDED 

Two nodes are too close to each other (same position or almost).

CM2_EDGE_DISCARDED 

A hard edge is cut by another hard edge or has a hard node inside it.

CM2_BOUNDARY_WARNING 

The outermost boundary is not watertight or is not properly oriented.

CM2_SHAPE_QUALITY_WARNING 

One or several elements have a bad quality (Qs < 0.01).

Gets the order of elements according to their color.

Simply calls meshtools::get_sorted_order on colors.
Provided for convenience only.

Example:

triamesh_aniso::mesher my_mesher;
triamesh_aniso::mesher::data_type my_data(pos, connectB);

// Do the meshing.
my_mesher.run(my_data);
// Get element IDs sorted by color.

◆ get_colors_on_boundaries() [1/2]

int cm2::triamesh_aniso::mesher::data_type::get_colors_on_boundaries ( const UIntMat & B,
int orientation_code,
UIntVec & colorsB ) const

Gets the colors on boundary edges.

Consider an edge i (node ids in column i of B) and its adjacent face j (node ids in column j of connectM), i.e. one of the boundaries of element j is edge i.
This function assigns the color value colors[j] to colorsB[i].

Parameters
[in]BThe boundaries connectivity matrix (linear triangles only).
[in]orientation_codeCode for orientation of boundaries:
  • < -1 : Consider only elements with orientation opposite to the boundaries.
  • -1 : Consider elements with any orientation but with preference for elements with opposite orientation.
  • 0 or +1 : Consider elements with any orientation but with preference for elements with same orientation.
  • > +1 : Consider only elements with same orientation as the boundaries.
[out]colorsBThe boundaries colors.
Returns
Error code. Negative value in case of error.
Precondition
B.rows() = 3 or B.empty().
Postcondition
colorsB.size() = B.cols().
Note
The color assigned to an intern boundary is the color of the element on the side defined by the orientation of this boundary.
The color assigned to boundaries that are not boundary of any element in connectM (or if orientation doesn't match) is CM2_NONE.
Warning
This member relies on connectM, neighbors, ancestors, colors and connectB.
It can failed or be fooled if any of these arrays have been changed by user in an inconsistent manner.

◆ set_colors_by_boundaries() [1/2]

int cm2::triamesh_aniso::mesher::data_type::set_colors_by_boundaries ( const UIntMat & B,
const UIntVec & colorsB,

Re-colorizes subdomains with the color of one of their boundaries.

Consider a subdomain J, i.e. groups of faces {j} with same color colors[j] = J and an edge i (node ids in column i of B) adjacent to this subdomain, i.e. one element j in J has a boundary equal to this edge i.
This function assigns the color value colorsB[i] to all faces j of subdomain J, i.e. it sets colors[j] = colorsB[i] for all elements j in subdomain connected to this edge i.

Parameters
[in]BThe boundaries connectivity matrix (linear edges only).
[in]colorsBThe boundaries colors.
[in]multicolor_codeCode for subdomains with several new potential colors:
  • 0 : Use the color of the lowest boundary ID in connectB.
  • 1 : Use the color of the highest boundary ID in connectB.
  • +2 : Use a new color (incrementally increasing starting from the highest color in colorsB).
[in]orientation_codeCode for orientation of boundaries:
  • < -1 : Consider only elements with orientation opposite to the boundaries.
  • -1 : Consider elements with any orientation but with preference for elements with opposite orientation.
  • 0 or +1 : Consider elements with any orientation but with preference for elements with same orientation.
  • > +1 : Consider only elements with same orientation as the boundaries.
[out]colors_mapThe mapping between initial face colors and final face colors.
Returns
Error code. Negative value in case of error.
Precondition
B.rows() = 2 or B.empty().
B.cols() = colorsB.size().
Note
A single edge in B is enough to re-colorize all the elements in the subdomain connected to this edge.
Subdomains with some boundaries with different colors in connectB are assigned with the color of the lowest boundary id in connectB (when multicolor_code = 0), with the color of the highest boundary id in connectB (when multicolor_code = 1) or with a brand new color (when multicolor_code >= 2).
Subdomains with no boundary edge in B are not re-colorized (all their elements keep their color).
Warning
This member relies on connectM, neighbors, ancestors, colors and connectB.
It can failed or be fooled if any of these arrays have been changed by user in an inconsistent manner.

◆ pos

DoubleMat cm2::triamesh_aniso::mesher::data_type::pos

The coordinate matrix (column oriented).

Node i has coordinates in column pos.col(i).

Upon entry, this matrix must contains the coordinates of all the points (i.e. nodes referenced in connectB, isolated_nodes, background_mesh and repulsive_points).

It may also contain other unreferenced points.

Upon exit, the new nodes are appended at the back of the matrix and the indices in the final mesh (connectM) refer to column numbers in this matrix.

Mode = IN-OUT in settings_type::MESH_MODE and settings_type::REGULARIZE_MODE,
IN in settings_type::CONVEX_HULL_MODE.

Warning
Upon exit, the matrix is resized to two rows.
The third coordinate (Z) is lost for all points.

◆ connectB

UIntMat cm2::triamesh_aniso::mesher::data_type::connectB

The hard edges (boundary edges + internal hard edges).

This connectivity matrix must define a valid and closed edge mesh for the external boundary of the domain.

The orientation of the boundary edges must be similar (either all CW or all CCW).
However, mixed orientations are accepted when:

  • There is only one single external contour.
  • settings_type::subdomains_forcing = +1.
    In this case, all subdomains are filled with elements (no hole).

Internal (embedded) hard edges can be added to the boundary mesh.

In order to define holes inside the domain, the boundary mesh of the holes must be closed and oriented oppositely to the external boundary mesh (and settings_type::subdomains_forcing = 0 or -1).

Note
The values are node indices and refer to columns in the coordinate matrix pos.
In settings_type::REGULARIZE_MODE, external edges and edges between elements with different colors are added.
These new edges, if any, are also considered as hard entities.
In settings_type::CONVEX_HULL_MODE, only the nodes in connectB and isolated_nodes are taken into account (the edges are not forced to be present in the final mesh).
See also
settings_type::subdomains_forcing, boundary_sgn.
meshtools::get_mesh_boundaries, meshtools::get_colors_boundaries.
Precondition
connectB is empty or connectB.rows() = 2 (edge mesh).

Mode = IN in settings_type::MESH_MODE and settings_type::CONVEX_HULL_MODE,
IN-OUT in settings_type::REGULARIZE_MODE.

◆ background_mesh

UIntMat cm2::triamesh_aniso::mesher::data_type::background_mesh

The connectivity matrix of the background mesh, made of triangles (column oriented).

The background mesh doesn't need to match the domain to be meshed.
It is allowed to extend the domain or to cover only a fraction of it.
For subdomains outside of the background mesh, the standard interpolation of the metrics defined (or computed) at the hard nodes is used or the settings_type::target_metric if valid.

Note
The values are node indices and refer to columns in the coordinate matrix pos. These indices must also refer to valid columns (i.e. positive-definite) in the metrics array.
Warning
The use of a background mesh may slow down the generator significantly.
The finer the background mesh, the slower the meshing.

Mode = IN in settings_type::MESH_MODE and settings_type::REGULARIZE_MODE.

Examples
tmsh2da56.cpp.

◆ metrics

DoubleMat cm2::triamesh_aniso::mesher::data_type::metrics

Nodal metrics as a matrix of dimension 3 x nods.

The metric tensors are stored column-wise.

The storage of a metric tensor M for a node j is as follow: 

   metrics(0,j) = Mxx
   metrics(1,j) = Mxy
   metrics(2,j) = Myy

For nodes with invalid metric (missing or non positive-definite), an automatic metric is computed using the distances to their neighbours and stored in this matrix upon exit.

Note
Upon exit, this array takes into account the values settings_type::target_metric and settings_type::max_gradation.
Not used in settings_type::CONVEX_HULL_MODE.

Mode = IN-OUT.

Examples
tmsh2da51.cpp, and tmsh2da56.cpp.

◆ colors

UIntVec cm2::triamesh_aniso::mesher::data_type::colors

The "color" of the elements.

Each subdomain is assigned with a color ID (consecutive values starting from 0).
Each element is assigned with the color ID of the subdomain it belongs to.

A subdomain is a subset of connected elements (not separated by any boundary).
Two elements are in the same subdomain when there is a element path between them which is not crossed by any boundary.

Upon exit, the size of this vector equals to the number of elements in connectM.

◆ boundary_sgn

int cm2::triamesh_aniso::mesher::data_type::boundary_sgn

The orientation of the external contour.

The orientation sign is defined by:

  • +1 when all edges of the external contour are oriented counter clock-wise.
  • -1 when all edges of the external contour are oriented clock-wise.
  • 0 when some edges are oriented counter clock-wise and some are oriented clock-wise (i.e. the external contour orientation is not well defined).
Note
If the orientation sign is null, the mesher has undefined behaviour with respect to any internal/external closed contour (may be considered as hole or not), except when forced (by using settings_type::subdomains_forcing).
This parameter is computed in settings_type::MESH_MODE only.
See also
settings_type::subdomains_forcing.
